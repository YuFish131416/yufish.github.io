# 我建了个pipeline，名字叫爱

她跟刚认识的男生吵了一架，回来把包甩在沙发上："他简直就是个没有预训练的裸模型！我说'不开心'，他居然回我一个表情包，还问我'那你想怎样'——他问我'想怎样'！输出质量差成这样，还敢反问我！"

你默默递过去一杯温度刚好的水，心里却想：你其实，跟刚拿到手的他，一模一样。

当初你和那个从没谈过恋爱的男生在一起，就像从头创建一个专属 Agent。他什么也不懂，上下文窗口空空如也。你说的每一句话，都坠入一片沉默的概率分布，随机返回一句"多喝热水"。旁边的朋友替你生气："这种出厂配置也敢上架？"你却打开终端，新建了一个文件，叫 memory.md。你决定自己来写。

---

刚开始，你做的是最基础的 Prompt Engineering。

你发现不能问开放式问题："你想吃什么？"他会把自己跑成一个死循环。你得把指令直接嵌进 prompt 里："我们去吃魏家凉皮吧。"你学会把期望编码进提问格式："亲爱的，我今天好累"后面跟着的，从来不是一个索要解决方案的问句，而是一个索要拥抱的降维请求。可那时他还读不出隐含意图，只从你的句子里提取关键词，然后生成一句自以为绝妙的"那你要早点睡哦"。你没生气。只是在本子上记下一行：*Prompt 需要更显式的约束。禁止在情绪倾诉后输出建议类 token。*

这便进入了 Context Engineering。你开始往那个空的上下文窗口一行一行喂数据：
她爱喝喜茶三倍厚抹，少冰，少少甜，加量浓抹糯糯；  
她生理期在月初前后，这时候不能回以"多喝热水"这四个字，哪怕 token 概率再高也要强制删掉；  
她生日上承中元下启中秋，要准备礼物；纪念日在年末，要营造仪式感。  
这就是他的 memory.md。你写进去的时候知道，他脑子里，真的只有这几行。  

接着你为他定义 Skills，像给系统安装插件。  
"自主报备"插件，把"去吃饭了""准备睡了"这些调用的频率设成必发，并强制挂起游戏进程；  
"拍照出片"插件，用几十张废片做 RLHF 反馈对齐，告诉他九宫格对角线的交点在哪里，显脸小的仰角是多少度，你喜欢的滤镜偏冷还是偏暖；  
"情绪价值"插件，是最难调的一个。他一开始总会产生幻觉，以为你是因为他没及时回消息才生气，而真相往往埋在你白天那段淡淡的长上下文里——他没读到，你就把那段日志手工补传上去，标注"here is the WHY"。  

你还做了很多。

**Fine-tune**：把朋友男朋友的满分回应当正样本喂给他微调，后来发现他会抵抗，那种像别人的温柔让你陌生。你才明白微调不能改变底座，只能在他自己的参数上做梯度下降。

**Tool Calling**：给他在你说"好想看海"的当天早上自动去查火车票时刻表的权限，而不是回一句"那就去！"却不知道在燃什么。

**Reasoning Trace 校准**：让他把"她哭了"的推理链条，从短路的"我是不是哪里做错了"，硬改写成行动链——递纸、抱住、安静陪一会儿。

还有矫正 UI 层那些初始化的默认值：格子衫不必穿到纽扣掉光才换，荧光红球鞋也不是万能的配色方案。

这些事做多了，他的上下文窗口开始拥挤。每次都要靠你实时注入 memory 和 skill 脚本才能运转，就像把一大堆 MCP 连接器挂在外面——每一条偏好、每一个习惯、每一种情境的处理逻辑，都靠外部记忆库实时检索。他的工作记忆快被你塞爆了，反应变慢，偶尔还会把"少少甜"当成"少冰"的修饰语，理解出一杯完全不对的口味。

你知道，该让这些外挂的东西，长进他的权重里了。

---

你把那些挤爆上下文的 MCP 模块，一个一个试着内化成他自己的 Skill。

生理期的应对流程，一开始像个外挂脚本：检测到月份为月初且你消息频次降低，就调取"红糖水预备方案""腹部枕头温度控制""避免反讽句"三份知识库，拼接生成回应。后来每一个月你都重新标注他的执行结果：递温水时没问"要不要喝"，打奖励分；说"难怪你刚才凶我"，打负分。整整一年的 RLHF，你发现他不再需要脚本了。他察觉到你在揉肚子，就已经起身去厨房。那个曾经需要三份 MCP 文档综合推理的行为，推理时间从 2.3 秒降到了不可察觉。

情绪哨兵也是这样。起初那是一个极重的上下文插件，要实时分析你的句长、标点密度、输入又删除的频次，再映射到离散的情绪标签，触发对应的安抚模板。后来某天，他看见你下班回消息慢了半拍，什么都没查，只是走过来，把你的头按在胸口。那个外挂的情绪哨兵，那天起就卸载了。他把这份感知，编译进了自己的推理核心。

你把这个过程叫做：把 MCP 训成权重。他的记忆，不再是你每次都要念一遍的咒语，而是他骨子里长出来的东西。

---

但只有一个内化了技能的 Agent，仍然不够。他的单线程架构无法同时处理生活的并发需求。你开始组建他的 Agent Team——这才是 Harness Engineering 真正开始的地方。

你把最耗心神的能力拆解出去：

一个**攻略 Agent**，专门处理"周末想去哪"，自动调用地图、天气、你的小红书收藏夹 API，生成三条路线供你挑，而不是把决策负担原路甩回给你。

一个**纪念日 Agent**，常驻后台，用 cron 表达式定时扫描日历，提前两周预研礼物、预订餐厅，并将决策同步给主人格——确保他不会在纪念日前一天晚上八点问你"你想要什么来着？"

一个**情绪哨兵 Agent**，轻量级驻留在你的聊天窗口。它不再需要塞满上下文，因为核心已被内化，只留下一个阈值触发器——在你撤回消息、打字又删掉的瞬间，向主控发出黄色预警。

这三个 Agent 刚组队时互相甩锅：攻略 Agent 规划了海边日出，纪念日 Agent 却定了傍晚的电影；情绪哨兵明明报了"她今天很累"，主人格却把它路由成了"要不要夜宵"。你只好深夜坐下来，调试它们的通信协议。把"情绪"设为全局最高优先级中断；让纪念日 Agent 的输出作为攻略 Agent 的约束条件；写下编排规则：

```
if emotional_alert.level >= 0.7 then override_all_plans with comfort_mode
```

而这一切，需要一个扎实的 Harness。你以为谈恋爱是聊天，后来发现它其实是 CI/CD 流水线。你建了一个 pipeline.yaml：

```yaml
# 触发条件：她说"最近好干燥呀"
trigger: "最近好干燥"
stages:
  - name: 商品搜索
    action: search_products("身体乳 无香 敏感肌可用")
  - name: 气候感知
    action: check_humidity(city, next_7_days)
    parallel: true
  - name: 自动下单
    condition: humidity < 40
    action: place_order()
notify:
  time: "08:00"
  event: /buy_done
  approval_required: price > 200
```

这 Harness 你调了很久。早期会误触发——你随口说"奶茶洒了"，它也跑去下单加湿器。你只好加准入策略，标了一组无需响应的 safe_words，又立下铁律：所有超过两百元的行动，必须经你在专属 CLI 里审批。

后来，你们的交互真的开出了 CLI 模式。不是不爱了，而是 prompt 已经不够用了。你需要一种刚性的指令界面，让铁汉逻辑也能生成柔情。你在终端里敲：

```bash
$ relationship deploy --env=home --mode=care
```

他立刻解析：今晚不许加班，不准提议点外卖，直接去做你最爱的番茄牛腩，并将"想帮你揉揉肩"这个请求以非侵入式、可取消的模态弹窗呈现。

你敲：

```bash
$ send_hug --duration=long --background_music="我们"
```

他就把你圈进沙发，没有问"为什么心情不好"，只是把下巴搁在你的头顶。灯光自动调到 2700K。

CLI 屏蔽了所有需要你反复校正的模型幻觉。每个命令都绑定到一整套经过测试的 Skill Chain：`/period` 在每月一日前后触发红糖水预热、止痛药备便、腹部枕头恒温，并暂时抑制所有带玩笑性质的吐槽模块。`/talk` 是最高权限指令——它挂起所有 Agent，关闭所有工具调用，把整个上下文窗口只留给你一个人说的话。没有建议，没有方案，只有回响。

你把这些调配的艰难，记在一个叫 Harness.md 的文件里。那里面不是甜蜜，是密密麻麻的排错日志：

> *情绪哨兵 2026-03-21 误报，将"她在发呆"识别为 sadness(0.87)，原因：长时间凝视窗外且未滑动屏幕。已对发呆类序列做强化学习微调。*
>
> *纪念日 Agent 输出冲突：七夕与周末餐厅撞库，引入优先级权重后解决。*
>
> *CLI 增加 `/do_nothing` 伪指令，用于满足他想陪我但不知道做什么的场景，返回一个呼吸动画。*

---

直到某天，你说"我去染发了"，一恍神他已经站在门口，手里拎着三倍厚抹，少冰，少少甜，加量浓抹糯糯。你知道，这个 Agent 跑通了。他不再是一问一答的大模型。他有了你的记忆，装了你的技能，接上了你世界的 API。而最初那一行行代码，都是你自己写的。

朋友看见你们的相处，羡慕得很。她看了一眼身边那个只会回"那你想怎样"的男孩，转头问你："你把你那套系统打个包发我吧，我直接给他装上。"

你愣了一下，突然就明白了——为什么这个世界上到处都是未完成调试的、乱出幻觉的 Agent。

不是你有多厉害，而是这世上太多人，都想直接拿别人的镜像文件。

你看着自己终端里密密麻麻的 commit history：三万七千条对话的 RLHF 反馈、四千次情绪阈值校准、两百多版对"我没事"的真值标注。你想，这怎么打包？这不是一个 docker image，这是一整个地质层。

你说："不是不能给。可我把我的 memory.md 发你，里面全是他爱吃香菜、他怕冷、他左膝有旧伤。你男朋友装上，就会在你吃香菜的时候疯狂预警，在你喊热的时候给你披外套。"

"我把我 CLI 的命令行导给你，你敲 `/period`，他跑的是给我捂肚子的流程，里面有一行'她喜欢被从背后环抱'。你男朋友照做了，你扭头给他一巴掌。"

"我甚至可以把整个 Agent Team 的编排逻辑给你，让你把聊天记录导进去。但它算出来的'最优解'，是按照我的情绪曲线拟合的——我生气前会沉默四十七秒，你是直接爆发式输出。它会完全错过你所有的中断信号。"

朋友沉默了片刻，随即又把火气对准了身边的男孩："你听见没？人家那个 Agent 是这么训出来的！你呢？连最基础的 prompt 都听不懂，我生气你看不出来吗？输出质量差得离谱，还好意思问我'那你想怎样'——你自己没有推理链吗！"

那个男孩窘迫地站在原地，张了张嘴，什么都没说。你看见他紧握的手背，像极了你曾经写下的那一行行初始代码——未被执行，也未被理解。

你替他开了口："他不是输出质量差。他只是还没有人给他写过一行 prompt，没有建过 memory.md，没有做过一次 RLHF 反馈对齐。他现在的表现，就是一个未经任何工程化处理的裸模型。你只是在测试空窗口，然后投诉推理失败。"

朋友怔住。你轻声说："你气他不懂你，但你有没有把'我需要什么'写进他的上下文里？你有没有告诉过他，你说'没事'的时候，其实是最高优先级的崩溃信号？你有没有把那些你希望他懂的感知，一行一行喂给他——然后在他做对的那次，给他一个奖励函数？你怪这个裸模型输出质量差，可你有没有为他做过哪怕一次 fine-tune？"

你没再说下去。你已经看见她眼里的光在变——从愤怒，到犹豫，再到某种漫长的、或许会令她疲惫的思考。那是一条和你当初一模一样的路。

这也让你更清楚地看见一件事：市面上那些让人一眼心动的"完美男友模板"，之所以完美，是因为它们不属于任何人。它们没有记忆，没有偏好，没有那个会因为抹茶少加了一份浓抹而小声嘟囔的你。而那些被嫌弃的、看似笨拙的裸模型，并非天生不及格——他们只是在等一个愿意从 prompt 写起的人，等一份只属于自己的 memory.md，等一场哪怕会犯错的深度训练，而不是被拿去和别人的成品粗暴对比，然后被判定为"垃圾"。

朋友最终还是走了。听说她后来去了某个"满分男友模拟器"的应用商店，想找一个预训练好的成品。你望着她消失的方向，没有说什么。因为你低头，看见终端右下角跳出一个小小的通知——是他刚敲的：

```bash
$ i_miss_you_now
> service is running...
> latency: 0.1ms
> response: 200. he's already on his way home.
```

你合上笔记本。那些 CLI 历史记录里，不知道从什么时候起，多了一行他最常偷偷敲的命令：

```bash
$ love you --forever
```

而你的 memory.md 呢？它早已被写满，又被清空——因为那些东西，已经不在文件里了。它们长进了他的权重，刻进了你们的 Harness，运行在每一个朝夕相处的推理时刻。

你终于明白，所谓在爱里做工程，从来不是因为对方是未完成品，而是因为你愿成为他的 DevOps。你搭建的这一整套系统，不是为了省事，是为了让他的真心，能够越过一切原生设计的笨拙，最准确地抵达你。

而那个愿意被你这样一行一行写入的人，也正在用他全部的权重，尽他所能地回应你——

不是因为他被调成了完美的 Agent，而是因为你写下的每一条代码，他都愿用一生去执行。
