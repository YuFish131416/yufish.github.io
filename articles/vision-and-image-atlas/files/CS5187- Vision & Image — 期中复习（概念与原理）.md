# Lec00：课程导论 — 计算机视觉是什么，为何困难

## 一、计算机视觉的目标

计算机视觉的核心目标是让机器**从图像和视频中感知并理解世界**。

具体来说，它试图回答以下问题：
- 这张图里有**什么物体**？（识别）
- 它们的**3D形状和空间关系**如何？（几何重建）
- 图中**正在发生什么事**？（场景理解）
- 能否让图像**变得更好**？（增强、去模糊、超分辨率）

> **一句话理解**：计算机视觉就是让计算机"看懂"图片背后的故事。

## 二、为什么计算机视觉很困难？

人类可以轻松看懂一张照片，但计算机面对的只是一个**数字矩阵**。让识别变困难的核心原因是：**同一物体在不同条件下看起来可以完全不同；不同物体在某些条件下看起来却很相似。**

具体挑战：

| 挑战 | 直觉解释 |
|------|----------|
| **视点变化** | 一只猫从正面看和从背面看，像素完全不同 |
| **光照变化** | 同一个房间白天和晚上拍出的图像差异巨大 |
| **尺度变化** | 远处的人和近处的人在图像中大小完全不同 |
| **类内变化** | "椅子"这个类别包含无数种长相不同的椅子 |
| **遮挡** | 物体的一部分被其他东西挡住，信息丢失 |
| **背景杂乱** | 物体和背景混在一起，难以分离 |
| **局部歧义** | 只看图像的一小块区域，可能有多种解释 |

> **关键理解**：这些挑战不是独立的，它们常常同时出现，使得问题指数级复杂。

## 三、人 vs 计算机

- 人类在"感知型"任务上仍然更鲁棒、更快、成本更低
- 但深度学习（尤其是近10年）使得计算机在越来越多的任务上接近甚至超越人类
- "什么是困难的"这个边界在不断被重新定义

## 四、课程结构概览

| 模块 | Lecture | 核心内容 |
|------|---------|----------|
| **低层视觉** | Lec01-04 | 图像处理、滤波、边缘检测、特征检测 |
| **几何与匹配** | Lec05-10 | 特征描述、变换、对齐、RANSAC、相机 |

# Lec01：图像与图像滤波 — 如何"加工"一张图片

## 一、图像到底是什么？

理解图像有三个层次：

| 层次 | 理解方式 | 说明 |
|------|----------|------|
| 物理层 | **光线的记录** | 相机/人眼捕捉光在空间中的传播并记录下来 |
| 数学层 | **二维函数** | 灰度图像可以看作 $f(x,y)$，输入坐标，输出亮度值 |
| 计算层 | **像素矩阵** | 数字图像就是一个二维整数数组，0 = 黑，255 = 白 |

> **核心理解**：数字图像是连续函数 $f(x,y)$ 经过**采样**（空间离散化）和**量化**（值域离散化）后得到的矩阵。这个"采样"的概念会在 Lec03 中成为重要话题。

## 二、什么是滤波？为什么需要滤波？

**滤波（Filtering）** = 根据每个像素的**局部邻域**信息，计算出一个新的像素值，生成新图像。

为什么要做这件事？
- **提取信息**：从图像中提取边缘、角点等有意义的结构
- **去除噪声**：消除拍摄中的随机干扰
- **增强图像**：让图像更清晰、更适合后续处理
- **CNN 的基础**：卷积神经网络的核心操作就是滤波

> **直觉**：滤波就像给图像戴上不同的"眼镜"——模糊眼镜让一切变柔和，锐化眼镜让细节跳出来，边缘眼镜只让轮廓留下来。

## 三、线性滤波与卷积核

**线性滤波**的做法：将每个像素替换为邻域像素的**加权和**。权重由一个小矩阵决定，这个小矩阵叫 **卷积核（kernel）**，也叫 mask 或 filter。

**操作过程**：把核"滑"过图像的每个位置，在每个位置做核与邻域的逐元素相乘再求和（类似"点积"）。

### 卷积 vs 互相关

| | 互相关（Cross-Correlation） | 卷积（Convolution） |
|---|---|---|
| 核的操作 | **不翻转**，直接滑动 | 核先**翻转 180°**，再滑动 |
| 直觉 | "模板匹配" | 严格数学定义的线性运算 |
| 实际使用 | 当核对称时两者等价 | 满足交换律和结合律 |

> **考试关键**：当核是对称的（如高斯核），卷积 = 互相关。只有非对称核时才有区别。课程中大多数滤波器是对称的，但要理解概念区别。

## 四、常见滤波器及其直觉

### ① 单位冲激核（Identity Kernel）
含义：中心为 1，其余为 0。输出 = 输入，什么都不做。

> **为什么重要**：它是构造锐化核的基础组件。

### ② 平移核（Shift Kernel）
含义：1 不在中心而在偏移位置。图像整体平移对应方向。

### ③ 均值滤波（Box/Mean Filter）
含义：所有权重相等（如 3×3 核每个元素都是 1/9）。

效果：每个像素变成邻域平均值 → **模糊**。
问题：边缘也被模糊了，而且会产生**振铃效应**（ringing）。

### ④ 高斯滤波（Gaussian Filter）⭐

**核心思想**：权重按高斯分布——中间大、边缘小，离中心越远贡献越少。

**关键性质（概念层面）**：
1. **低通滤波器**：去掉高频成分（噪声、细节），保留低频（整体结构）
2. **旋转对称**：各方向效果相同
3. **σ 控制模糊程度**：σ 越大，核越宽，模糊越强，保留的结构越"大尺度"
4. **自卷积性**：高斯 * 高斯 = 更大的高斯（用 σ 卷两次 = 用 $\sqrt{2}\sigma$ 卷一次）
5. **没有振铃效应**：相比均值滤波更"干净"

> **高斯 vs 均值的核心区别**：均值滤波器给所有邻居相同权重（"一刀切"），高斯给近邻更多权重、远邻更少权重（"近亲远疏"）。结果是高斯模糊更自然，边缘保持更好。

### ⑤ 锐化滤波（Sharpening）⭐

**核心思想**：锐化 = 原图 + 高频细节。

推导直觉：
1. 原图 = 低频部分 + 高频部分
2. 模糊图 ≈ 低频部分
3. 高频细节 = 原图 − 模糊图
4. **锐化图 = 原图 + α × 高频细节**（α 控制锐化强度）

> **一句话理解**：锐化就是"把模糊拿走的东西加倍放回来"。提取高频的操作又叫**高通滤波（high-pass filter）**。

## 五、非线性操作不能用线性滤波实现

**阈值化（Thresholding）**：像素值 > 阈值就变白，否则变黑。

> 它**不能**用线性滤波器实现，因为线性操作只是加权求和，无法表达"大于则为 1、否则为 0"这样的分段逻辑。这说明并非所有图像操作都是线性的。

## 六、边界处理

当核滑到图像边缘时，邻域有一部分"超出"了图像范围，怎么办？

| 方式 | 做法 | 特点 |
|------|------|------|
| **补零** | 边界外当作 0 | 简单，但边缘会变暗 |
| **镜像** | 边界处反射像素 | 效果自然，常用 |
| **循环** | 图像首尾相接 | 适合频域处理 |
| **裁剪** | 不计算边界像素，输出尺寸变小 | 避免了假象 |

> **考试提示**：理解每种方式的思路和适用场景即可，不需要记公式。

# Lec02：边缘检测 — 从像素中找到"轮廓"

## 一、什么是边缘？边缘从哪里来？

**边缘** = 图像中亮度**急剧变化**的位置。

> **直觉**：把图像想象成一片地形，亮度是高度。边缘就是地形中的"悬崖"——高度突然变化的地方。

边缘的物理成因：

| 成因 | 例子 |
|------|------|
| **深度不连续** | 物体和背景的分界线 |
| **表面法线不连续** | 物体形状折角处 |
| **表面颜色/纹理不连续** | 不同材质的交界处 |
| **光照不连续** | 阴影的边界 |

> **为什么重要**：边缘提取将图像从"像素阵列"压缩为"曲线集合"，保留了最重要的场景结构信息，是后续很多算法的起点。

## 二、用导数找边缘

**核心思想**：亮度急剧变化 = 亮度函数的**导数很大**。

- **一阶导数的极值**（峰值）→ 边缘位置
- **二阶导数的过零点** → 边缘位置（另一种判据）

### 图像梯度

对于二维图像，导数推广为**梯度**：
- **梯度方向**：亮度增大最快的方向（**垂直于边缘方向**）
- **梯度幅值**：边缘的"强度"——值越大，边缘越明显

> **关键理解**：梯度方向 ⊥ 边缘方向。如果边缘是水平的，梯度指向竖直方向。

### 数字图像如何求导？

连续函数可以直接求导，但数字图像是离散的。做法：
- 用**有限差分（Finite Difference）** 近似导数：$f'(x) \approx f(x+1) - f(x)$
- 这等价于与核 `[-1, 1]` 做卷积 — 又回到了 Lec01 的滤波操作！

## 三、噪声问题与"先平滑再求导"

**问题**：噪声会让导数到处都很大，产生大量虚假边缘。

**解决方案**：先用高斯滤波平滑图像，再求导。

> **巧妙之处**（利用卷积结合律）：$\frac{d}{dx}(f * G_\sigma) = f * \frac{d}{dx}G_\sigma$
>
> 可以预先计算**高斯的导数（Derivative of Gaussian, DoG）** 核，然后只做一次卷积，既平滑了又求了导。

## 四、Sobel 算子

Sobel 算子是**高斯导数的一种常用近似**，是一个 3×3 的核。

> **直觉**：它比简单的 `[-1, 1]` 差分更鲁棒，因为它在垂直于求导方向上也做了平滑（类似高斯的思想）。

## 五、Canny 边缘检测器 ⭐

Canny 是最经典的边缘检测方法，完整流程分四步：

### Step 1：平滑 + 求导
用高斯导数滤波器得到 x 和 y 方向的梯度。

### Step 2：计算梯度幅值和方向
幅值 = 边缘强度；方向 = 边缘走向（垂直于梯度）。

### Step 3：非极大值抑制（NMS）
**目的**：把"粗"边缘变成"细线"。

**做法**：沿梯度方向看，只保留局部最大值的那个像素，其余压为 0。

> **直觉**：就像在山脊上只保留最高点——如果你不是沿梯度方向的局部最高，你就不算真正的边缘。

### Step 4：双阈值 + 边缘连接（Hysteresis Thresholding）
设定两个阈值 $T_{high}$ 和 $T_{low}$：
- 幅值 > $T_{high}$：**强边缘**，直接保留
- $T_{low}$ < 幅值 < $T_{high}$：**弱边缘**，只有与强边缘相连时才保留
- 幅值 < $T_{low}$：丢弃

> **为什么不用单阈值？** 单阈值会导致边缘断裂。高阈值保证精确，低阈值保证完整，双阈值兼顾两者。

## 六、σ 的尺度效应

- **大 σ** → 平滑程度更强 → 检测大尺度（粗略）边缘
- **小 σ** → 保留更多细节 → 检测精细边缘

**尺度空间性质（Scale Space）**：
- 随着 σ 增大，边缘位置可能**偏移**
- 两条边缘可能**合并**
- 但一条边缘**不会分裂**成两条

> **考试要点**：Canny 的四个步骤及每步的作用；双阈值的意义；σ 对检测结果的影响。

# Lec03：图像重采样与插值 — 缩小放大时会发生什么

## 一、下采样为什么会出问题？

最朴素的缩小方法是"每隔一个像素扔一个"（subsampling）。但这样做图像会变得很难看——出现锯齿、摩尔纹、奇怪的条纹。这就是**混叠（Aliasing）**。

> **经典例子**：棋盘格不断缩小时，朴素下采样会出现奇怪的斜纹；正确的做法是先模糊再缩小，棋盘格最终变成均匀的灰色（黑白的平均）。

## 二、混叠（Aliasing）的本质

**奈奎斯特采样定理**：要无失真地采样一个信号，采样率必须 ≥ 信号最高频率的 2 倍。

> **直觉**：每个"波浪"至少需要 2 个采样点才能被正确记录。如果采样点不够密，高频信号会"伪装"成低频信号——这就是混叠。

**2 × 最高频率** = **奈奎斯特频率（Nyquist frequency）**

- **日常例子**：电影中直升机桨叶看起来反转（时间混叠）、车轮倒转（wagon-wheel effect）

## 三、正确的下采样：先模糊再采样

**解决方案**：下采样前先用**高斯低通滤波器**平滑图像，去掉那些"采样率不够记录"的高频成分，然后再丢弃像素。

> **一句话理解**：先把你记不住的信息扔掉（模糊），再缩小，就不会记错（混叠）了。

## 四、高斯金字塔（Gaussian Pyramid）⭐

反复执行"高斯模糊 + 2 倍下采样"，得到一系列逐级变小的图像：

```
原图 → 1/2 → 1/4 → 1/8 → ...
```

**关键性质**：
- **存储代价**：整个金字塔只比原图多 1/3 的空间（几何级数求和 $= N \times \frac{4}{3}$）
- **多尺度表示**：底层有细节，顶层有整体结构

**应用**：
- 图形学中的 **Mipmap**（纹理映射）
- 模板匹配（在不同层搜索可以处理尺度变化）
- 小波变换的前身

## 五、上采样与插值

放大图像时需要"填"出图像中没有的像素值，这就是**插值（Interpolation）**。

**核心思想**：数字图像是连续函数的离散采样 → 用**重建滤波器（reconstruction filter）**"猜回"连续函数 → 在新分辨率上重新采样。

| 插值方法 | 重建核形状 | 效果 | 特点 |
|----------|-----------|------|------|
| **最近邻** | 矩形（方块） | 块状、锯齿 | 最简单、最快 |
| **双线性** | 三角形（tent） | 平滑但略模糊 | 常用、效果适中 |
| **双三次** | 三次样条 | 更锐利 | 计算稍复杂 |
| **理想重建** | sinc 函数 | 理论最优 | 实际无法精确实现 |

> **考试提示**：理解这几种方法的直觉差异即可——最近邻是"取最近值"，双线性是"周围四点加权平均"，双三次考虑更多邻居所以更精细。

## 六、现代超分辨率

- **RAISR**：基于机器学习的快速超分辨率
- **多帧超分辨率**：利用多张有微小位移的照片合成更高分辨率图像（如手机的 burst 拍摄 + 手抖带来的亚像素偏移）

> 这些是拓展知识，了解思路即可。

# Lec04：Harris 角点检测 — 找到图像中"独一无二"的点

## 一、为什么需要局部特征？

很多视觉任务（全景拼接、3D 重建、物体识别、AR 跟踪）都需要在不同图像之间建立**对应关系**。流程是：

1. **检测**：在每张图中找到"兴趣点"
2. **描述**：给每个兴趣点一个"身份证"（特征向量）
3. **匹配**：比较两张图的"身份证"，找到对应

> 局部特征的优势：**局部性**（对遮挡鲁棒）、**数量多**（一张图可以有数百个）、**区分力强**、**可实时计算**。

## 二、什么样的点适合做特征？

**核心直觉**：一个好的特征点必须是"独一无二的"——在图像中能明确定位，不会跟其他点混淆。

### 三种区域类型

| 区域类型 | 窗口移动时的变化 | 能做特征吗？ |
|----------|------------------|--------------|
| **平坦区域** | 任意方向变化都很小 | ❌ 到处都一样，无法定位 |
| **边缘** | 沿边缘方向变化小，垂直方向变化大 | ❌ 沿边缘滑动时分不清位置 |
| **角点** | **任意方向都有大变化** | ✅ 唯一可定位 |

> **直觉**：把一个小窗口盖在图像上，向任意方向移动。如果窗口内的图像都会明显改变，那就是角点。

## 三、Harris 角点检测的思路

### Step 1：定义"变化"
窗口平移 $(u,v)$ 后与原来的**像素差平方和（SSD）**：
$$E(u,v) = \sum_{(x,y) \in W} [I(x+u, y+v) - I(x,y)]^2$$

### Step 2：用泰勒展开简化
当移动很小时（小运动假设），$E(u,v)$ 可以近似为一个**二次型**：

$$E(u,v) \approx \begin{bmatrix} u & v \end{bmatrix} H \begin{bmatrix} u \\ v \end{bmatrix}$$

其中 $H$ 是 **二阶矩矩阵（Second Moment Matrix）**，由窗口内各点的梯度信息构成。

> **关键理解**：$H$ 矩阵编码了"这个点的邻域在各方向上变化有多快"的信息。

### Step 3：用特征值判断区域类型 ⭐

$H$ 是一个 2×2 对称矩阵，有两个特征值 $\lambda_1, \lambda_2$。

| $\lambda_1, \lambda_2$ 的关系 | 几何含义 | 结论 |
|------|----------|------|
| 都很小 | 各方向变化都小 | **平坦区域** |
| 一大一小 | 只有一个方向变化大 | **边缘** |
| **都很大且接近** | **各方向变化都大** | **角点** ✅ |

**椭圆可视化**：$H$ 对应一个椭圆：
- 椭圆轴方向 = 特征向量（变化的方向）
- 轴长度 ∝ $\lambda^{-1/2}$（特征值越大，轴越短）
- 角点 → 椭圆接近圆形；边缘 → 椭圆极扁

> ⭐ **与特征值计算的联系**：这里就是课程中特征值最重要的应用。考试可能要求你对一个 2×2 的 $H$ 矩阵算出特征值，然后判断该点是角点、边缘还是平坦区域。详见 `eigenvalues.md`。

## 四、Harris 响应函数

直接计算特征值开销较大，Harris 提出了一个**不需要显式求特征值**的近似方法：

$$R = \det(H) - k \cdot \text{trace}(H)^2 = \lambda_1\lambda_2 - k(\lambda_1+\lambda_2)^2$$

- $R$ 大 → 角点
- $R$ 负且绝对值大 → 边缘  
- $|R|$ 小 → 平坦区域

> $k$ 通常取 0.04~0.06。另一个常用定义是 $R = \frac{\det(H)}{\text{trace}(H)}$。

## 五、Harris 检测完整流程

1. 计算图像梯度 $I_x, I_y$
2. 计算梯度乘积 $I_x^2, I_y^2, I_xI_y$
3. **高斯加权**这些乘积（而非简单窗口求和）→ 得到 $H$ 矩阵
4. 计算响应函数 $R$
5. 阈值化 + **非极大值抑制** → 最终角点

> **为什么用高斯加权？** 和 Lec01 一样的道理——距离中心越近的像素越重要。简单方框窗口效果不好。

## 六、为什么不能只看"梯度大"？

一个常见误区：有人以为"x 和 y 方向梯度都大的地方就是角点"。**错！** 对角线上的边缘在 x 和 y 方向梯度都大，但它仍然是边缘。Harris 的方法通过矩阵特征值分析，能正确区分这种情况。

> **考试要点**：三种区域的特征值关系；$H$ 矩阵的含义；Harris 响应函数；特征值与椭圆的几何可视化。

# Lec05：特征不变性 — 在变化的图像中找到"不变的"特征

## 一、不变性 vs 等变性

这两个概念是理解特征检测器性能的关键：

| 概念 | 含义 | 直觉 |
|------|------|------|
| **不变性（Invariance）** | 图像变换后，特征**检测结果不变** | 亮度变了，角点还在同一位置 |
| **等变性（Equivariance）** | 图像变换后，特征**随之做同样的变换** | 图像旋转 90°，角点也旋转 90° |

> 很多文献中"不变性"泛指两者。**等变性**有时也叫 covariance。

## 二、Harris 检测器的不变性分析

| 变换类型 | Harris 表现 | 原因 |
|----------|------------|------|
| **平移** | ✅ 等变 | 梯度和窗口都随之平移，角点也跟着移 |
| **旋转** | ✅ 等变 | 椭圆整体旋转，但特征值（形状）不变 |
| **亮度平移** $I→I+b$ | ✅ 不变 | Harris 只用导数，常数偏移导数为 0 |
| **亮度缩放** $I→aI$ | ⚠️ 部分 | 响应值被缩放，阈值可能需要调整 |
| **尺度变化** | ❌ 不具备 | 放大后角点可能变成边缘 |

> **核心问题**：Harris 对尺度变化**完全没辙**。这引出了下面的尺度不变检测。

## 三、尺度不变检测 — 如何让特征"跨尺度"

**思路**：在**位置和尺度**的三维空间 $(x, y, \text{scale})$ 中同时找响应的极值点。

**实现**：
1. 在图像的**高斯金字塔**（多个尺度）上都计算响应
2. 找到某个尺度下响应最大的点 → 该尺度称为**特征尺度（characteristic scale）**
3. 每个特征输出 $(x, y, s)$：位置 + 尺度

## 四、LoG 与 DoG 斑点检测器 ⭐

### LoG（Laplacian of Gaussian）
**含义**：对高斯滤波后的图像求二阶导数（拉普拉斯算子）。

**核心作用**：**斑点（Blob）检测器**——对图像中的圆形区域有强烈响应。

**特征尺度**= LoG 响应取最大值时对应的 σ（直觉：σ 与斑点大小匹配时，响应最强）。

### DoG（Difference of Gaussians）
两个不同 σ 的高斯之差：$\text{DoG} = G_{\sigma_1} - G_{\sigma_2} \approx \text{LoG}$

> **为什么用 DoG 代替 LoG？** 计算效率更高，高斯金字塔本身就在做减法。SIFT 就是用 DoG。

### 检测流程
多尺度计算 → 在 (x, y, scale) 三维空间中找极值 → 输出每个斑点的位置和尺度

| 比较 | Harris | LoG/DoG |
|------|--------|---------|
| 检测对象 | 角点 | 斑点 |
| 尺度不变 | ❌ | ✅ |
| 旋转等变 | ✅ | ✅ |

> **考试要点**：不变性 vs 等变性的区别；Harris 对各种变换的表现；尺度不变检测的核心思路（在位置+尺度空间找极值）；LoG 与 DoG 的关系。

# Lec06：特征描述子与匹配 — 给特征点一个"身份证"

## 一、描述子的两大要求

检测到兴趣点后，需要**描述**每个点的外观，以便在不同图像间匹配。好的描述子需要：

| 要求 | 含义 |
|------|------|
| **不变性** | 图像旋转、缩放、光照变化后，同一个点的描述子应该（基本）相同 |
| **可区分性** | 不同点的描述子应该差异显著，否则匹配就会混乱 |

> **矛盾**：太不变就会丧失区分力（极端：描述子恒为 0，完全不变但无法匹配）。好的描述子在两者间取得平衡。

## 二、如何实现旋转不变？

**核心做法**：先找到特征点邻域的**主方向**（比如梯度最强的方向），然后把邻域**旋转到标准朝向**后再描述。

这样无论图像怎么旋转，描述出来的结果都一样。

## 三、SIFT 描述子 ⭐

**全称**：Scale-Invariant Feature Transform（David Lowe, UBC）

**构建过程**：
1. 取兴趣点周围 **16×16** 窗口
2. 计算每个像素的**梯度方向**，弱梯度的像素丢弃
3. 将窗口分成 **4×4 网格**（共 16 个单元格）
4. 每个格子统计 **8 个方向**的梯度直方图
5. 按主方向旋转以实现旋转不变
6. 拼接所有格子 → **128 维向量**（16 格 × 8 方向）

**SIFT 的鲁棒性**：
- 可处理约 60° 的视角变化
- 可处理白天到黑夜的光照变化
- 计算速度适中（中等图像 < 1 秒）

## 四、其他描述子（了解即可）

| 描述子 | 核心思想 | 特点 |
|--------|----------|------|
| **MOPS** | 40×40 窗口缩到 8×8 + 旋转 + 强度归一化 | 简单，但区分力有限 |
| **HOG** | 局部梯度方向直方图 + block 归一化 | 行人检测经典方法 |
| **ORB** | FAST 角点 + BRIEF 二进制描述（256 位） | **实时**，设备端 SLAM |
| **LIFT** | 深度学习端到端学习检测+描述 | 数据驱动 |

## 五、特征匹配

### 简单方法：L2 距离
直接比较两个描述子向量的欧氏距离。**问题**：模棱两可的匹配也可能距离很小。

### 更好的方法：比率距离（Ratio Distance）⭐
$$\text{ratio} = \frac{\|f_1 - f_2\|}{\|f_1 - f_2'\|}$$

- $f_2$ = 最佳匹配，$f_2'$ = 第二佳匹配
- **正确匹配**：最佳远优于第二佳 → ratio 小
- **错误匹配**：最佳和第二佳差不多 → ratio ≈ 1

> **关键**：比率距离**不改变最佳匹配是谁**，但改变了距离值的分布，使得用阈值筛选正确/错误匹配更容易。

## 六、匹配性能评估：ROC 曲线

设定某个阈值后，所有距离低于阈值的匹配被保留：
- **真阳性（TP）**：保留的且正确的匹配
- **假阳性（FP）**：保留的但错误的匹配
- **真阳性率 TPR** = TP / 所有正确匹配总数
- **假阳性率 FPR** = FP / 所有错误匹配总数

**ROC 曲线**：扫描所有阈值，画出（FPR, TPR）曲线。
- 曲线越靠左上角 → 描述子/匹配方法越好
- **AUC**（曲线下面积）越接近 1 越好

> **考试要点**：SIFT 的构建步骤和 128 维的来源；比率距离的思想和优势；TP/FP/TPR/FPR 的定义；ROC 曲线的含义。

# Lec07：图像变换与变形 — 图像之间的几何关系

## 一、滤波 vs 变形

| 操作 | 改变的是 | 直觉 |
|------|----------|------|
| **滤波（Filtering）** | 像素的**值** | 把图变亮、变模糊 |
| **变形（Warping）** | 像素的**位置** | 把图旋转、拉伸、扭曲 |

## 二、2×2 矩阵能做什么？不能做什么？

2×2 矩阵可以表示：**缩放、旋转、剪切、镜像**。

**不能**表示：**平移**！因为平移需要加一个常数项，不是纯线性运算。

**线性变换的四个性质**：
1. 原点映射到原点
2. 直线映射到直线
3. 平行线保持平行
4. 比例关系保持

## 三、齐次坐标 — 让平移也能用矩阵乘法 ⭐

**问题**：平移不是线性变换，无法用 2×2 矩阵表示。

**解决**：给坐标加一个维度！$(x, y) → (x, y, 1)$

这样平移就变成了 3×3 矩阵乘法：
$$\begin{bmatrix} x+t_x \\ y+t_y \\ 1 \end{bmatrix} = \begin{bmatrix} 1 & 0 & t_x \\ 0 & 1 & t_y \\ 0 & 0 & 1 \end{bmatrix} \begin{bmatrix} x \\ y \\ 1 \end{bmatrix}$$

> **从齐次坐标变回去**：$(x, y, w) → (x/w, y/w)$。当 w = 1 时就是普通坐标。

## 四、变换的层次结构 ⭐

这三种变换形成**嵌套关系**，从简单到复杂：

### ① 仿射变换（Affine）— 6 个自由度

最后一行固定为 $[0\ 0\ 1]$，包含：平移 + 旋转 + 缩放 + 剪切。

**关键性质**：
- 直线 → 直线
- **平行线保持平行** ← 和单应性的核心区别
- 比例关系保持

### ② 单应性 / 投影变换（Homography）— 8 个自由度

最后一行可以是任意值。$H$ 和 $2H$ 等价（只差一个尺度因子）。

**关键性质**：
- 直线 → 直线
- **平行线不一定保持平行**（铁轨在照片中会交汇！）
- 比例关系不保持
- 可以表示透视效果

**变换嵌套**：
```
线性变换 ⊂ 仿射变换 ⊂ 投影变换（单应性）
```

> **什么时候用单应性？** 两个相机只有旋转差异（纯旋转），或者场景是一个平面。这在全景拼接中很常见。

## 五、图像变形的实现

### 正向变形（Forward Warping）
对每个源像素，算出它在目标图中的位置。
**问题**：目标位置可能不是整数 → 需要"分配"到周围像素；可能产生**空洞**。

### 逆向变形（Inverse Warping）⭐ 推荐
对目标图中的每个像素，反查它在源图中的位置 $T^{-1}(x', y')$，然后**插值**。
**优点**：不会产生空洞，实际中更常用。

> **考试要点**：仿射 vs 单应性的性质区别（平行线是否保持）；齐次坐标的作用；变换的自由度；正向/逆向变形的区别。

# Lec08：图像对齐与最小二乘 — 如何用匹配点算出变换

## 一、图像对齐的总体流程

```
Step 1: 两幅图各自检测特征点
Step 2: 匹配两幅图的特征
Step 3: 用匹配点对计算变换 T（最小二乘法）
```

## 二、最小二乘法 — 核心思想 ⭐

**问题**：有 $n$ 对匹配点 $(p_i, p_i')$，想找一个变换 $T$ 使所有点尽量对齐。但通常点对数 > 未知数个数（超定系统），无法精确满足所有约束。

**做法**：最小化所有匹配点的**残差平方和**：
$$\min_T \sum_i \|p_i' - T(p_i)\|^2$$

### 简单例子：平移
- 未知数：$(t_x, t_y)$，2 个
- 每对匹配给 2 个方程
- 最小二乘解 = **所有位移向量的平均值**

### 仿射变换
- 6 个未知数 → 至少需要 **3 对点**
- 写成矩阵形式 $A\mathbf{p} = \mathbf{b}$，用正则方程 $\mathbf{p} = (A^TA)^{-1}A^T\mathbf{b}$ 求解

### 单应性（Homography）
- 8 个自由度 → 至少需要 **4 对点**
- 由于分母非线性，改写为 $A\mathbf{h} = \mathbf{0}$ 的形式（齐次线性方程组）
- 解 = $A^TA$ **最小特征值对应的特征向量**（通过 SVD 求解）

> ⭐ **与特征值的联系**：求单应性的过程就是求 $A^TA$ 的最小特征值和对应的特征向量。这是特征值分解在本课程中的又一重要应用。

## 三、离群点问题

最小二乘对**离群点（outliers）极其敏感**——一个错误匹配就可能严重拉偏整个结果。

> 解决方案：RANSAC（下一讲）。

> **考试要点**：各变换需要的最少匹配点数（平移 1 对，仿射 3 对，单应性 4 对）；最小二乘的正则方程；单应性用 SVD 求解的原因。

# Lec09：RANSAC — 在有错误匹配时鲁棒地估计变换

## 一、为什么最小二乘不够？

特征匹配中不可避免有错误匹配（outliers）。最小二乘把所有点都"信任"了，一个 outlier 就可能把结果拉偏。

> **托尔斯泰式理解（Efros 的玩笑）**："所有正确匹配都是相似的；每个错误匹配各有各的错法。"

## 二、RANSAC 核心思想 ⭐

**"假设-验证"策略**：不用所有点拟合，而是随机抽一小批点拟合模型，然后看其他点是否"同意"。

## 三、RANSAC 算法流程

```
重复 N 次：
  1. 随机选 s 个匹配点对（最小样本集）
  2. 用这 s 个点拟合模型（如：4 对点 → 单应性）
  3. 对所有其他点，计算在此模型下的残差
  4. 统计残差 < 阈值 ε 的点数（inliers）
选取 inlier 数最多的模型
[后处理] 用所有 inliers 重新做一次最小二乘 → 最终模型
```

## 四、关键参数

| 参数 | 含义 | 如何设定 |
|------|------|----------|
| **s** | 最小样本数 | 由模型自由度决定（平移 1，仿射 3，单应性 4） |
| **ε** | inlier 阈值 | 通常是噪声标准差的 2~3 倍 |
| **N** | 迭代轮数 | 由 outlier 比例 $e$ 和期望成功概率 $p$ 决定 |

### 迭代轮数公式 ⭐

$$N \geq \frac{\log(1-p)}{\log(1-(1-e)^s)}$$

> **直觉**：每次随机取 $s$ 个点全部为 inlier 的概率是 $(1-e)^s$。$N$ 次中至少成功一次的概率要 ≥ $p$。

**关键规律**：$s$ 越大或 outlier 比例越高，需要的轮数**急剧增加**。

| | outlier 20% | outlier 50% |
|---|---|---|
| s=2 | 5 轮 | 17 轮 |
| s=4 | 9 轮 | 72 轮 |
| s=8 | 26 轮 | 1177 轮 |

## 五、RANSAC 的优缺点

| 优点 | 缺点 |
|------|------|
| 简单通用 | 参数需要调 |
| 实践效果好 | outlier > 50% 时失效 |
| 自动区分 inlier/outlier | 极端情况迭代次数过多 |

## 六、全景拼接完整流程

```
特征检测（Harris/DoG）
  → 描述子（SIFT）
    → 匹配（比率距离）
      → RANSAC 估计单应性
        → 逆向变形 + 插值
          → 合并图像 = 全景图
```

> **考试要点**：RANSAC 的四步流程；迭代轮数公式及影响因素；各模型的最小样本数 $s$；与最小二乘的关系（RANSAC 找 inlier 集合，最终仍用最小二乘拟合）。

# Lec10：相机模型 — 3D 世界如何变成 2D 图像

## 一、为什么需要理解相机？

之前我们一直在 2D 图像上工作。但现实世界是 3D 的——理解相机如何把 3D 场景投影到 2D 图像，是进入**三维重建**和**立体视觉**的基础。

## 二、针孔相机模型

**核心思想**：所有光线通过一个小孔（针孔），投影到对面的胶片上。

> **直觉**：像墨子时代就发现的小孔成像——一个暗箱，一端开小孔，另一端会出现倒立的影像。

### 透视投影公式

3D 点 $(x, y, z)$ 投影到图像平面上的 $(x', y')$：

$$x' = f \cdot \frac{x}{z}, \quad y' = f \cdot \frac{y}{z}$$

> **关键**：除以 $z$（深度）这个操作使得**近大远小**——这就是透视效果。

### 用齐次坐标表示
虽然除以 $z$ 是非线性的，但齐次坐标下可以用矩阵乘法实现投影，最后再做归一化（除以 $w$）。

## 三、相机参数的分解 ⭐

完整的投影过程分为两部分：

```
3D世界坐标 --[外参]--> 3D相机坐标 --[内参]--> 2D像素坐标
```

### 外参（Extrinsics）— 6 个参数
描述相机在世界中的**位置和朝向**：
- **旋转 $R$**（3×3 矩阵，3 个参数）：相机的朝向
- **平移 $\mathbf{c}$**（3×1 向量，3 个参数）：相机的位置

> 外参回答的问题是："相机在哪里？朝哪个方向看？"

### 内参（Intrinsics）— 最多 5 个参数
描述相机自身的光学属性：

| 参数 | 含义 | 直觉 |
|------|------|------|
| **$f$** | 焦距（像素为单位） | 控制"缩放"——$f$ 大 = 望远，$f$ 小 = 广角 |
| **$c_x, c_y$** | 主点 | 光轴与图像平面的交点，通常在图像中心附近 |
| **$\alpha$** | 宽高比 | 像素是否正方形（通常 = 1） |
| **$s$** | 倾斜系数 | 像素是否平行四边形（通常 = 0） |

> 实际中常简化为 3 个参数：$f, c_x, c_y$（假设像素正方形、无倾斜）。

## 四、正交投影 vs 透视投影

| | 透视投影 | 正交投影 |
|---|---|---|
| 公式 | $(x/z, y/z)$ | $(x, y)$ — 直接丢掉 $z$ |
| 效果 | 近大远小 | 没有透视效果 |
| 适用场景 | 一般场景 | 物体离相机极远（望远镜头近似） |

## 五、透视效果与消失点

- **平行线在透视投影下汇聚于消失点（Vanishing Point）**
  - 每组平行线有自己的消失点
  - 但平行于图像平面的平行线仍保持平行

> **日常例子**：铁轨看起来在远处"交汇"，就是消失点效应。

## 六、径向畸变

真实镜头并不完美，会导致**径向畸变**：
- **桶形畸变（Barrel）**：图像向外鼓
- **枕形畸变（Pincushion）**：图像向内缩

> 这些畸变在镜头边缘最明显，可以通过多项式模型校正。

> **考试要点**：透视投影公式及"除以 z"的含义；内参/外参各包含什么以及参数个数；焦距的直觉意义；正交投影的概念；消失点。

# Lec13：双目立体视觉 — 从两张图中恢复深度

## 一、为什么需要两个"眼睛"？

单张图像上的一个像素点，对应 3D 空间中的**一条射线**——你无法确定点在射线上的哪个位置（深度丢失了）。

但如果有**两张从不同位置拍的图**，同一个 3D 点在两张图上各对应一条射线。两条射线**在唯一的一个 3D 点上相交**，于是深度就确定了。

> **一句话理解**：一只眼睛看到方向，两只眼睛看到距离——这就是双目立体视觉的本质。

## 二、核心几何：对极几何与视差

### 对极几何（Epipolar Geometry）
给定两个已标定的相机，左图中一个点在右图中的**对应搜索范围不是整个图像**，而是一条线——称为**对极线（Epipolar Line）**。

> 在**校正后的立体对（Rectified Stereo Pair）**中，对极线就是水平的扫描线——左图的第 $y$ 行和右图的第 $y$ 行对应。

### 视差（Disparity）
左图像素在 $(x_1, y)$，它在右图中匹配到 $(x_2, y)$，则：

$$\text{disparity} = x_1 - x_2$$

### 视差与深度的关系 ⭐
$$Z = \frac{f \cdot B}{d}$$

其中 $f$ 是焦距，$B$ 是基线（两个相机之间的距离），$d$ 是视差。

> **关键直觉**：**视差与深度成反比**。近处的物体在两张图之间"移动"更多（视差大），远处的物体几乎不移动（视差小）。你可以用手指实验——把手指放在眼前，交替闭左右眼，手指跳动很明显；放远后跳动变小。

## 三、立体匹配算法

### 基本思路
在左图中取一个像素（或窗口），在右图的**同一行**中搜索最相似的位置：

```
For each 行 (epipolar line):
  For each 左图像素:
    在右图同行中搜索，找到匹配代价最小的位置
    → 该位置差即为 disparity
```

### 匹配代价函数

| 方法 | 全称 | 特点 |
|------|------|------|
| **SSD** | Sum of Squared Differences | 最简单，对亮度变化敏感 |
| **SAD** | Sum of Absolute Differences | 计算量较小 |
| **ZNCC** | Zero-mean Normalized Cross Correlation | 对光照变化鲁棒（归一化消除了亮度偏移） |

### 窗口大小的权衡 ⭐

| 窗口大小 | 优点 | 缺点 |
|----------|------|------|
| **小窗口** | 保留细节 | 噪声多 |
| **大窗口** | 结果平滑，噪声少 | 丢失边界细节 |

> 自适应窗口（Adaptive Window）可以兼顾两者。

## 四、立体匹配作为能量最小化 ⭐

简单的逐像素匹配只考虑了"每个像素自己匹配好不好"，却忽略了**相邻像素的深度应该是连续的**。更好的框架是把匹配看作一个**能量最小化问题**：

$$E(d) = \underbrace{\sum_{(x,y)} C(x, y, d(x,y))}_{\text{匹配代价 (data term)}} + \underbrace{\lambda \sum_{(p,q)\in \mathcal{N}} V(d_p - d_q)}_{\text{平滑代价 (smoothness term)}}$$

- **匹配代价**：每个像素在其视差下与对应像素的相似度
- **平滑代价**：鼓励相邻像素有相近的视差值

### 平滑代价 $V$ 的选择
- **L1 距离**：$V(d) = |d|$ — 允许缓慢的深度变化
- **Potts 模型**：$V(d) = \mathbf{1}(d \neq 0)$ — 只惩罚"有没有跳变"，不管跳多少

> $\lambda$ 控制平滑与匹配的平衡：$\lambda$ 太大 → 全局一个深度；$\lambda$ 太小 → 回到贪心逐像素匹配。

## 五、完整的立体重建流程

```
Step 1: 标定相机（获取内参 + 外参）
Step 2: 校正图像对（Rectification — 使对极线变为水平线）
Step 3: 计算视差图（Disparity Map）
Step 4: 由视差公式估计深度  Z = f·B/d
```

### 常见误差来源

| 误差来源 | 说明 |
|----------|------|
| 标定误差 | 内参/外参不准 |
| 分辨率低 | 像素级匹配精度有限 |
| 遮挡 | 一幅图中可见的区域在另一幅中被遮挡，无法匹配 |
| 亮度不一致 | 镜面反射等导致两图中同一点亮度不同 |
| 纹理贫乏区域 | 没有纹理的区域（白墙等）无法可靠匹配 |
| 重复纹理 | 重复图案导致多处都匹配得很好 |

## 六、主动深度感知

被动双目方法在纹理贫乏区域容易失败。**主动方法**通过主动投射已知图案来解决这个问题：

| 方法 | 核心原理 | 代表设备 |
|------|----------|----------|
| **结构光（Structured Light）** | 投射已知编码图案到物体上，简化匹配问题 | Kinect、iPhone Face ID |
| **飞行时间（Time-of-Flight）** | 测量光（红外）从发出到返回的时间 | ToF 相机 |
| **激光扫描** | 投射激光条并扫描，光学三角测量 | LiDAR |

> **考试要点**：视差的定义与 $Z = fB/d$ 反比关系；对极线是什么（搜索在一条线上而非整幅图）；窗口大小的权衡；能量最小化框架中 data term 与 smoothness term 各自的意义；误差来源；主动 vs 被动深度感知的区别。

# Lec14：多视图立体 — 从多张图到完整 3D 模型

## 一、从双目到多视图

| | 双目立体 | 多视图立体（MVS） |
|---|---|---|
| **输入** | 2 幅图像 | 多幅已标定图像 |
| **输出** | 视差图（深度图） | 完整的 3D 模型 |
| **基线** | 固定 | 可以选择最优基线 |

## 二、多视图的优势

- **更强的匹配信号**：多个邻居图像同时匹配，比只用两张更可靠
- **选择最优邻居**：对每个参考图像，可以选择最合适的若干邻居来匹配
- **完整重建**：每帧算一个深度图，再把所有深度图融合为完整的 3D 模型

## 三、基线选择的影响 ⭐

| 基线状况 | 问题 |
|----------|------|
| **基线过小** | 三角测量角度小，深度估计精度低 |
| **基线过大** | 两张图中同一表面看起来差异太大，匹配困难 |

> **多基线立体（Multi-baseline Stereo）**解决方案：不选一个基线，而是同时使用多个基线。计算每对的 SSD 再累加（称为 **SSSD**），兼顾精度与鲁棒性。

## 四、平面扫描立体（Plane-Sweep Stereo）⭐

这是 MVS 中一种核心且直观的方法。

### 核心思想
假设场景中的某个点位于参考相机前方的某个**深度平面**上。如果假设正确，将邻居图像通过**单应性变换**重投影到该平面后，应该与参考图像一致（photo-consistent）；如果深度假设错误，重投影结果会不一致。

### 算法流程

```
1. 在参考相机前方设置一系列深度假设平面（从近到远）
2. 对每个深度假设：
   将所有邻居图像通过单应性 H 重投影到该深度平面
3. 比较重投影结果与参考图像的相似度 → 存入代价体积
4. 从代价体积中求解深度图
```

### 代价体积（Cost Volume）
一个三维张量，维度为 $\text{height} \times \text{width} \times \text{depth\_hypotheses}$：
- 位置 $(x, y, d)$ 存储的是：在深度假设 $d$ 下，像素 $(x, y)$ 的匹配代价
- 最终深度图 = 对每个像素取匹配代价最小的深度假设

> 也可以使用更复杂的求解器（如 Belief Propagation、Graph Cuts）来结合平滑项求解，类似 Lec13 的能量最小化。

## 五、深度图融合

每个参考图像生成一张深度图后，需要将多张深度图**融合**为一个完整的 3D 模型。

## 六、NeRF 简介

**NeRF（Neural Radiance Fields）**是一种现代的隐式 3D 表示方法：
- 将场景表示为一个从 $(x, y, z, \theta, \phi)$ → $(RGB, \sigma)$ 的神经网络
  - 输入：空间位置 + 观察方向
  - 输出：该点的颜色和密度（透明度）
- 使用**体积渲染（Volume Rendering）**从任意视角合成图像
- 给定多张已标定照片 → 训练网络 → 能从**任意新视角**渲染场景

> **考试要点**：双目 vs 多视图立体的区别；基线过大/过小各自的问题；平面扫描立体的核心思想和流程；代价体积的含义；NeRF 的基本概念（输入/输出是什么）。

# 附录 A：易混淆概念对比速查

| 概念 A | 概念 B | 核心区别 |
|--------|--------|----------|
| **卷积（Convolution）** | **互相关（Cross-Correlation）** | 卷积需要先翻转核（flip），互相关直接滑动。对称核两者结果相同 |
| **线性滤波** | **非线性滤波（如 Median）** | 线性 = 加权求和（满足叠加性）；中值等无法用加权和表示 |
| **高斯滤波** | **均值滤波** | 高斯给中心更大权重，保留更多细节；均值等权，模糊更均匀 |
| **Sobel 算子** | **Canny 边缘检测** | Sobel 仅计算梯度；Canny 是完整流程（平滑→梯度→NMS→双阈值） |
| **Aliasing（混叠）** | **Blur（模糊）** | 混叠是采样率不足导致的不可逆失真；模糊是故意低通滤波以防止混叠 |
| **高斯金字塔** | **拉普拉斯金字塔** | 高斯 = 每层模糊降采样；拉普拉斯 = 相邻层的差分（保留细节/带通） |
| **不变性（Invariance）** | **等变性（Equivariance）** | 不变 = 输出不变（如描述子对旋转不变）；等变 = 输出随输入同样变化（如检测位置随平移移动） |
| **Harris 角点** | **LoG/DoG blob** | Harris 检测角点（依赖特征值比值）；LoG/DoG 检测 blob（在尺度空间中找极值） |
| **SIFT 描述子** | **HOG 描述子** | SIFT = 关键点邻域的 4×4×8 梯度直方图；HOG = 整幅图的密集网格梯度直方图 |
| **仿射变换** | **单应性变换** | 仿射 = 6 DOF，保持平行线；单应性 = 8 DOF，不保持平行线，但保持直线 |
| **前向映射（Forward Warping）** | **反向映射（Inverse Warping）** | 前向：从源像素找目标位置（可能有空洞）；反向：从目标位置找源像素（无空洞） |
| **最小二乘** | **RANSAC** | 最小二乘受 outlier 影响严重；RANSAC 先随机采样剔除 outlier 再拟合 |
| **内参（Intrinsics）** | **外参（Extrinsics）** | 内参 = 相机自身属性（焦距等，最多5个参数）；外参 = 相机在世界中的位姿（R, t，6个参数） |
| **透视投影** | **正交投影** | 透视 = 除以 z（近大远小）；正交 = 直接丢掉 z（无透视效果，望远近似） |
| **视差（Disparity）** | **深度（Depth）** | 两者成反比（$Z = fB/d$）；视差是像素位移量，深度是 3D 距离 |
| **双目立体** | **多视图立体（MVS）** | 双目 = 2幅图→视差图；MVS = 多幅图→完整3D模型 |
| **被动立体** | **主动深度（结构光/ToF）** | 被动 = 纯光学匹配（textureless 区域困难）；主动 = 投射已知图案辅助 |

# 附录 B：特征值在课程中的出现地图

特征值（Eigenvalue）和特征向量不只是线性代数知识——它们在本课程中**反复出现**且扮演不同角色：

## 1. Harris 角点检测（Lec04）⭐ 最核心

**矩阵**：$H = \sum \begin{bmatrix} I_x^2 & I_xI_y \\ I_xI_y & I_y^2 \end{bmatrix}$（梯度外积的求和）

**特征值的角色**：$H$ 的两个特征值 $\lambda_1, \lambda_2$ 决定了局部区域的类型：

| $\lambda_1, \lambda_2$ | 含义 |
|--------------------------|------|
| 两个都小 | 平坦区域 — 无纹理 |
| 一大一小 | 边缘 — 沿一个方向有强梯度 |
| 两个都大 | 角点 — 两个方向都有强梯度 ✓ |

> Harris 的 $R = \det(H) - k \cdot \text{tr}(H)^2$ 巧妙地避免了直接计算特征值，但其本质就是在判断特征值的分布。

## 2. PCA / 主成分分析（Lec06 MOPS 描述子背景）

降维时，协方差矩阵的**特征值**表示各方向的方差大小，**特征向量**表示主方向。选最大的若干特征值对应的方向做投影。

## 3. SVD 求解单应性（Lec08）

最小二乘的法方程 $A^T A \mathbf{h} = 0$ 涉及 $A^T A$ 的特征值分解（等价于对 $A$ 做 SVD）。**最小特征值对应的特征向量**就是最优解。

## 4. 特征值计算方法（考试可考）

对 $2 \times 2$ 矩阵 $\begin{bmatrix} a & b \\ c & d \end{bmatrix}$：

$$\lambda = \frac{(a+d) \pm \sqrt{(a+d)^2 - 4(ad - bc)}}{2} = \frac{\text{tr} \pm \sqrt{\text{tr}^2 - 4\det}}{2}$$

> 详细的计算例题见 `eigenvalues.md`。
