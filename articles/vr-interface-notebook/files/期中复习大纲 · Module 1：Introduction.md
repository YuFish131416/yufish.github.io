## 一、虚拟现实的定义（What is Virtual Reality）

### 1.1 核心定义

> VR 是一种 **人机交互界面（Human-Computer Interface）**，包含两个核心要素：
>
> *   **实时仿真（Real-time Simulation）**
> *   **多感官通道的交互（Multi-sensory Interaction）**

所支持的感知通道（Sensorial Modalities）包括：

*   视觉（Visual）
*   听觉（Auditory）
*   触觉（Tactile）
*   嗅觉（Smell）
*   味觉（Taste）

### 1.2 VR 的四个层级（Levels of VR）

| 层级 | 名称                     | 特点                   |
| -- | ---------------------- | -------------------- |
| 1  | **被动型（Passive）**       | 用户无控制权，内容固定（如看电影、读书） |
| 2  | **探索型（Exploratory）**   | 可在虚拟世界中移动探索，但不能修改内容  |
| 3  | **交互型（Interactive）**   | 可与虚拟世界中的物体交互（如抓取、移动） |
| 4  | **协作型（Collaborative）** | 多用户在虚拟世界中协同完成任务      |


> ⚠️ **考点提示**：本课程主要关注**第2、3、4层级**。

## 二、人机交互界面（Human Computer Interfaces）

### 2.1 基本概念

*   HCI 关注如何从用户处**捕获交互指令（Input）**，经计算机分析后再**将反馈输出（Output）给用户**
*   目标：让人与计算机的交互方式类似人与人之间的多感官互动
*   典型感知循环：`see → analyse → action`

### 2.2 输入/输出设备分类

*   **输入设备（Input Devices）**：用于感知（Sensing），捕获用户行为信号
*   **输出设备（Output Devices）**：用于响应（Actions），向用户传递反馈
*   部分设备兼具输入输出功能

### 2.3 各感知通道的技术挑战

#### 视觉（Vision）

*   计算机**呈现**虚拟世界（图像渲染）：已有大量研究，技术相对成熟
*   计算机**理解**真实世界（计算机视觉）：难度远大，传统方法依赖像素，难以恢复高层语义
*   深度学习的兴起推动了视觉语义恢复的进展（如自动驾驶场景理解）

#### 听觉（Sound）

*   音频可传达信息，也能帮助感知环境（如判断车辆方向与距离）
*   同视觉一样：**生成声音比识别声音更容易**
*   深度学习显著提升了语音识别性能

#### 手势（Gesture）

*   手势携带丰富信息，可通过3D图像来呈现
*   硬件方案：电子手套（Electronic Gloves）捕获手部姿态

#### 触觉（Tactile Sensing）

*   指力觉（Sense of Force）
*   **输入**：压力传感器检测用户施加的力
*   **输出**：机械或液压装置向用户施力
*   关键缺陷：此类设备具有**侵入性（Intrusive）**——即使不工作，用户也能感受到其存在

#### 其他感知（其他感官）

*   **嗅觉**：某些气味可模拟，但消除困难；部分气味难以检测
*   在大多数VR应用中很少被考虑

## 三、VR硬件发展历程（Progress in VR Hardware）

### 3.1 历史时间线总览

```
1838          ~1960s         1968           1985-88      Late 1980s     1992    1995    2012-2018
  |              |             |               |             |            |       |         |
立体镜        HMD原型      Sutherland      NASA系列       VPL系列      CAVE   Virtual  VR爆发期
(Stereoscopes) (Heilig)   "Ultimate Display"  VIVED/VIEW   EyePhones          Boy    Oculus等
```

### 3.2 关键里程碑详解

#### 立体镜（Stereoscopes，1838）

*   发明者：Wheatstone、Brewster 等
*   早期通过双目视差原理实现立体视觉

#### Morton Heilig 的 HMD（1960）

*   名称：**Simulation Mask（仿真面罩）**
*   包含：3D立体视频（2个电视管）、立体声音（2个耳机）、气味（空气喷嘴）

#### Sensorama（1962，Morton Heilig）

*   **首款虚拟现实视频机器**
*   功能：3D视频 + 运动 + 立体声 + 气味 + 风效（风扇）
*   经典体验：模拟骑摩托车穿越纽约，感受路面颠簸和餐厅气味

#### Sutherland 的 HMD（1968）

*   名称：**"Ultimate Display（终极显示器）"**
*   技术：使用阴极射线管（CRT），因其过重而通过天花板机械臂支撑
*   创新：使用**计算机生成的场景**（而非预录视频）
*   局限：当时计算机仅能渲染 200–400 个多边形，刷新率 20 Hz

#### 军用飞行模拟器（Military Flight Simulators）

*   早期：基于预录视频，价格昂贵
*   军队动机：创建数字模拟器训练飞行员，更换机型仅需软件升级
*   飞行头盔与现代模拟器均在此阶段建立，但相关成果属机密，未公开发表

#### 英国航空航天（British Aerospace，1987–1990s）

*   **VECTA（Virtual Cockpit 1987）**：完全沉浸式HMD；缺陷：无法看到自己的手
*   **RAVECTA**：视频透视HMD，采用蓝屏（色键技术）合成户外环境

#### NASA VIVED（1985）

*   目的：训练宇航员体验太空条件
*   技术：拆解 Sony Watchman 电视机，提取 LCD 屏幕制作 HMD

#### NASA VIEW（1988）

*   在 VIVED 基础上进化
*   新增：感知手套（Sensing Gloves）、可操控最多4个3D虚拟声源的硬件
*   声音锁定功能：即使头部转动，声源位置保持不变
*   渲染：从线框（Wireframe）升级为表面渲染（Surface Rendering）

#### EyePhones，VPL（1980年代末）

*   **首款商用HMD**
*   规格：LCD（320×240），立体图像
*   问题：画面模糊、价格 US$11,000、重量 2.4kg

#### CAVE（1992）

*   多面大型投影屏幕围绕用户，提供沉浸式环境
*   无需佩戴HMD

#### Nintendo 系列

*   **Power Gloves**：超声波传感器测腕部位置、导电油墨弯曲传感器测手指弯曲 → 因游戏不足，1993年停产
*   **Virtual Boy（1995）**：游戏主机级VR，商业失败

#### VPL 数据手套（Data Gloves）

*   基于光纤传感器的感知手套
*   问题：价格昂贵、手型适配困难 → VPL 1992年倒闭

#### VR 工具包（VR Toolkits）

*   早期：WorldToolKit (WTK)、Virtual Reality Toolkit (VRT3)（1993）
*   现代：主要基于游戏引擎，如 **Unreal Engine** 和 **Unity**

#### VR复兴（2012–2018，VR Explosion）

*   背景：早期系统昂贵、渲染质量差、分辨率低 → 市场萎缩

*   复兴驱动力：

    *   PC与图形硬件进步
    *   HMD技术进步（高分辨率、轻量化、低成本，借助手机屏幕技术）
    *   传感器与设备技术进步
    *   开放游戏引擎支持VR开发

*   代表产品：Oculus、Sony PlayStation VR、HTC Vive、Samsung Gear VR、Microsoft HoloLens 2

## 四、VR系统类型（Types of VR Systems）

### 4.1 四大类型总览

| 类型                       | 描述                          |
| ------------------------ | --------------------------- |
| 沉浸式VR（Immersive VR）      | 完全将用户沉浸在仿真环境中               |
| 非沉浸式VR（Non-immersive VR） | 通过普通显示器，也称"Limited Reality" |
| 增强现实（Augmented VR / AR）  | 在真实世界叠加虚拟信息                 |
| 远程呈现（Telepresence）       | 通过远程机器人感知和操控远端环境            |


> **注**：沉浸式与非沉浸式通常合称"VR"；远程呈现与机器人学密切相关

### 4.2 沉浸式VR（Immersive VR）

*   核心硬件：**HMD + 头部追踪系统**，提供360°虚拟视野

*   头部追踪器检测头部位置/方向，驱动图形系统渲染对应视角画面

*   其他实现方式：

    *   多面大型投影屏幕（如CAVE系统）
    *   物理环境仿真（飞行模拟舱、驾驶模拟器）

### 4.3 非沉浸式VR（Non-immersive VR）

*   无360°视野，用户通过**普通电脑显示器**观看

*   低成本（使用传统显示器）

*   可选附加设备：**立体眼镜（Stereo Glasses）**

    *   原理：镜片可编程控制透光，与显示器左右眼图像同步交替开闭
    *   大多数3D电视也采用此技术

*   **裸眼立体显示（Stereoscopic Displays）**：无需眼镜，通过视差屏障（Parallax Barrier）分离左右眼画面，但目前市场接受度不高

*   关键特性：用户只有在看屏幕时才"身处"虚拟世界

### 4.4 增强现实（Augmented VR / AR）

*   通过**透明眼镜**将数字图形/动画/视频叠加于真实世界之上

*   两种主要显示形式：

    1.  **双镜片式眼镜**：小型投影仪将视频图像投射到镜片上
    2.  **飞行头盔式**：投影仪附在头盔上，将环境信息投射到透明护目镜（Visor）

*   移动端AR：手机摄像头捕捉真实场景，实时叠加合成虚拟物体后显示

### 4.5 远程呈现（Telepresence）

*   允许用户可视化并操控**远端机器人**完成任务

*   工作原理：双向通信

    *   **远端 → 用户**：摄像头、麦克风等传感器信号传回控制室，通过HMD渲染
    *   **用户 → 远端**：用户佩戴传感器（头部追踪、电子手套），动作信号传送到远端机器人

*   典型应用场景：

    *   太空探索（Space Exploration）
    *   远程手术（Remote Surgery）
    *   危险作业（Danger Operations，如核事故处理）

## 五、位置传感器（Position Sensors）

> 在虚拟世界中探索或交互，计算机必须捕捉用户的物理坐标。

### 5.1 各类追踪方法对比

| 追踪方式                           | 原理                          | 优点        | 缺点              |
| ------------------------------ | --------------------------- | --------- | --------------- |
| **2D鼠标**                       | 相对位移检测                      | 廉价、易用     | 难用于3D环境；只返回相对位移 |
| **机械追踪器（Mechanical Tracker）**  | 手柄连接机械臂，运动产生位置信号            | 快速、精确     | 运动范围受机械臂长度限制    |
| **超声波追踪器（Ultrasonic Tracker）** | 固定发射器 + 可动接收器，比较信号到达时间差确定位置 | 廉价        | 分辨率低、易受噪声干扰     |
| **磁场追踪器（Magnetic Tracker）**    | 固定磁场发射器 + 目标处磁传感器           | 精度较高，使用广泛 | 受磁场干扰和大金属物体影响   |

## 六、典型沉浸式VR系统架构（Typical Immersive System）

### 6.1 硬件组成

```
┌─────────────────────────────────────────────────┐
│               典型沉浸式VR系统                   │
│                                                 │
│  [HMD]           [电子手套]        [图形工作站]  │
│  • 双LCD屏        • 弯曲传感器      • 生成立体图像│
│  • 光学镜片       • 磁传感器        • 实时渲染    │
│  • 双扬声器                                      │
│                                                 │
│  [磁场追踪系统]                                  │
│  • 磁场发射器（固定）                             │
│  • 磁传感器（附在HMD + 手套）                     │
└─────────────────────────────────────────────────┘
```

### 6.2 工作流程

1.  **视觉输出**：2图形工作站设置两个虚拟摄像机（对应双眼位置），分别渲染左右眼画面，发送到HMD的两块LCD屏 → 实现立体视觉
2.  **头部追踪**：HMD上的磁传感器实时上报头部位姿 → 更新渲染视角
3.  **手部追踪**：手套上的磁传感器追踪手的3D位置 + 弯曲传感器捕捉手势 → 实现虚拟物体交互

## 七、延迟问题（Latency）

### 7.1 延迟的定义

**总系统延迟（Total System Lag）**：从用户头部发生移动，到屏幕上显示的画面更新为对应视角所经历的**总时间差**。

### 7.2 延迟的来源（各阶段延迟叠加）

```
头部移动
  → ① 传感器信号生成延迟
  → ② 追踪系统处理信号、计算新位姿延迟
  → ③ 计算机接收信息并更新数据库延迟
  → ④ 图像生成器使用更新后数据库渲染新帧延迟
  → 屏幕画面更新  （= Total System Lag）
```

### 7.3 延迟的影响

*   视觉感知与大脑预期之间产生**不一致（Discrepancy）**
*   导致用户产生**不适感（Uncomfortable Feeling）**，严重时引起晕动症（VR Sickness）
