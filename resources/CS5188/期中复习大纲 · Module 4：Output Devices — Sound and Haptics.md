## 一、声音基础（Sound Properties）

*   声音是通过介质传播的**压力波（Pressure Wave）**
*   声速公式：

<!---->

```
c = √(K / ρ)
```

| 符号  | 含义               |
| --- | ---------------- |
| `K` | 刚度系数（弹性介质对形变的阻力） |
| `ρ` | 介质密度             |


*   空气中声速约为 **343 m/s（767 英里/小时）**

*   声音具有普通波的特性：

    *   **反射（Reflection）**：声波弹回
    *   **折射（Refraction）**：进入不同密度介质时角度改变
    *   **衍射（Diffraction）**：绕过障碍物弯曲

***

### 1.1 人类听觉系统（Human Auditory System）

*   **耳廓（Pinna）**：反射声波，放大某些频率、衰减另一些，为大脑判断声源方向提供信息
*   **耳道（Ear Canal）**：放大 **3–12 kHz** 范围内的声音
*   **中耳（Middle Ear）**：由精细骨骼构成，放大来自鼓膜的声压
*   **内耳（Inner Ear）**：将声压转换为大脑信号

**可听频率范围**：**20 Hz – 20 kHz**

> ⚠️ **考点提示**：人耳对 **3–4 kHz** 的声音最为敏感；对极低频或极高频声音不敏感。

***

## 二、3D 声音（3D Sound）

*   **立体声（Stereo Sound）**：声音固定，不随头部运动而改变
*   **3D 声音（3D Sound）**：声音随用户**头部位置和朝向**动态更新

***

### 2.1 声源定位（Sound Localization）

人类通过三类线索确定声源的三维位置：

| 线索类型                     | 说明       |
| ------------------------ | -------- |
| **方位线索（Azimuth Cues）**   | 确定声源水平角度 |
| **仰角线索（Elevation Cues）** | 确定声源垂直角度 |
| **距离线索（Distance Cues）**  | 确定声源距离   |


#### 方位线索（Azimuth Cues）

**耳间时间差（Interaural Time Difference, ITD）：**

*   声音以固定速度在空气中传播，先到达距声源较近的耳朵
*   两耳接收到同一声音的**时间差 Δt** 即为 ITD
*   **低频声音**主要依靠 ITD 判断方向（声源不太远时）
*   **低于 250 Hz** 时，室内混响（reverberation）干扰 ITD 估算 → 大脑转而关注声音**瞬态起始**，忽略低频稳态信息

**耳间强度差（Interaural Intensity Difference, IID）：**

*   较近的耳朵感知到更高的声音强度
*   此现象称为**头影效应（Head-Shadow Effect）**
*   对**高频声音**更为明显

#### 仰角线索（Elevation Cues）

*   若将耳朵简单建模为"孔洞"，某些空间位置的时间差和强度差完全相同 → 产生**前后感知混淆（Front-Back Confusion）**
*   实际上，耳廓会对声音进行反射，直达声与耳廓反射声相互叠加 → 某些频率放大，某些衰减
*   声音随**仰角**变化，大脑据此判断垂直方向

#### 距离线索（Distance Cues）

*   大脑结合对声源的**先验知识**（如低语声、警报声）与**感知响度**来估算距离
*   **运动视差（Motion Parallax）**：用户平移头部时，声源方向随之改变，用于估算距离

***

### 2.2 头部相关传递函数（Head-Related Transfer Function, HRTF）

**定义：** 声源发出的声音与到达耳朵的声音之间的关系，用**傅里叶变换**表示。

**测量方法：**

1.  在声源位置放置麦克风
2.  在内耳放置微型麦克风
3.  对比两者，计算传递函数

**关键公式：**

```
Yᵢ = Hᵢ ⊗ S
```

其中 `S` 为输入声源，`Hᵢ` 为用户 i 的 HRTF，`⊗` 为卷积运算符。

*   每个人的 HRTF 独一无二（因耳廓和躯干几何形状各异）
*   一旦确定 HRTF，可对任意输入声音进行变换（**卷积**），通过耳机播放，令用户感知到真实空间中的声音位置

**空间识别误差对比（Front-Back Reversals）：**

| 情况        | 误差率 |
| --------- | --- |
| 使用真实声音    | 5%  |
| 使用自身 HRTF | 10% |
| 使用他人 HRTF | 30% |


> ⚠️ **考点提示**：使用他人 HRTF 会导致空间识别率显著下降。

***

### 2.3 Convolvotron 系统

*   首款虚拟 3D 声音生成系统，由 **Crystal River Engineering** 在 1988 年为 NASA 开发

*   工作流程：

    1.  头部追踪器数据发送至主机
    2.  通过查找表计算左右耳的 HRTF
    3.  对每个声源分别进行卷积 → 合并后经 D/A 转换器发送至耳机

***

### 2.4 基于扬声器的 3D 声音（Speaker-Based 3D Sound）

*   PC 扬声器位于显示器左右两侧
*   用户头部须处于**甜区（Sweet Spot Zone）**内
*   与耳机不同，扬声器**无法为每只耳朵隔离声音** → 每只耳朵接收到两个扬声器的混合声音

**计算公式：**

```
Yleft  = Hl,l ⊗ Sleft  + Hl,r ⊗ Sright
Yright = Hr,l ⊗ Sleft  + Hr,r ⊗ Sright
```

*   `Yleft`、`Yright` 为期望的双耳声音（已知）
*   将 H 矩阵求逆 × Y，即可求出 `Sleft`、`Sright`（扬声器应播放的声音）

## 三、触觉反馈（Haptic Feedback）

触觉反馈分两类：

| 类型                                 | 说明                           |
| ---------------------------------- | ---------------------------- |
| **触觉反馈（Touch / Tactile Feedback）** | 实时传递接触面几何形状、粗糙度、温度           |
| **力反馈（Force Feedback）**            | 实时传递物体表面属性、重量、惯性；可主动阻止用户接触运动 |


***

### 3.1 人类触觉感知（Human Haptic Sensing）

皮肤有四种触觉感受器：

| 类型        | 位置 | 感受器名称                            |
| --------- | -- | -------------------------------- |
| **表面感受器** | 浅层 | Meissner 小体（Meissner Corpuscles） |
| **表面感受器** | 浅层 | Merkel 盘（Merkel Disks）           |
| **深层感受器** | 深层 | Pacinian 小体（Pacinian Corpuscles） |
| **深层感受器** | 深层 | Ruffini 小体（Ruffini Corpuscles）   |


#### 慢适应感受器（Slow-Adapting Sensors）

*   **Merkel + Ruffini**
*   对低频刺激（**0–10 Hz**）敏感
*   在持续力刺激下保持稳定放电
*   用于检测**准稳态**（边缘、皮肤拉伸、静态力）

#### 快适应感受器（Fast-Adapting Sensors）

*   **Meissner + Pacinian**
*   对高频刺激（**50–300 Hz**）敏感
*   主要感知接触力的**变化**
*   放电率下降极快 → 持续稳定的力会变得无法被察觉

#### 感知分辨率（Sensing Resolution）

| 身体部位           | 最小可分辨距离 |
| -------------- | ------- |
| 指尖（Fingertips） | 2.5 mm  |
| 手掌（Palm）       | 11 mm   |


*   皮肤机械感受器的**连续性阈值**仅为 **5 ms**，远小于眼睛（25 ms）

***

#### 温度感知（Temperature Sensing）

*   由**温度感受器（Thermoreceptors）**和**伤害感受器（Nociceptors）**共同感知
*   **温度感受器**：位于表皮（冷感）或真皮（热感）
*   **伤害感受器**：对 **-15°C 至 45°C** 以外的温度敏感，触发痛觉响应

***

#### 本体感觉（Proprioception）

*   对自身身体位置和运动的感知
*   由神经末梢、Pacinian 和 Ruffini 小体在**骨骼关节处**检测
*   本体感觉分辨率：

| 关节            | 分辨率（度）    |
| ------------- | --------- |
| 髋关节（Hip）      | 0.2° （最高） |
| 肩关节（Shoulder） | 0.8°      |
| 手指（Finger）    | 2.5°      |
| 脚趾（Toe）       | 6.1° （最低） |


> 髋关节和肩关节分辨率最高，反映了身体对肢端误差最小化的需求。

***

### 3.2 触觉反馈接口（Tactile Feedback Interfaces）

#### 触觉鼠标（Tactile Mouse）

*   普通交互时用户须注视屏幕确认操作是否正确
*   触觉反馈通过**额外线索**响应用户动作

**iFeel Mouse（Logitech）：**

*   内置电气执行器，可振动鼠标外壳（Vibrotactile Feedback）

***

#### CyberTouch 手套

*   在 CyberGlove 基础上加装 **6 个振动触觉执行器**
*   每个执行器：塑料胶囊内含直流电机，电机轴有**偏心配重** → 旋转时产生振动

***

#### 温度反馈手套（Temperature Feedback Gloves）

*   帮助用户感知虚拟物体的热学特性（温度、热导率）
*   高导热材料（如钢）握持时产生**冷感**
*   **热执行器（Thermal Actuator）**：热电热泵，由 N 型和 P 型半导体夹在陶瓷绝缘体之间构成
*   施加直流电流 → P/N 电荷向热沉板移动 → 热源板降温、热沉板升温
*   电流越大，两板温差越大

**C\&M Research 温度感知系统工作流程：**

```
目标温度（Temp Command）
  → PID 控制器（比较目标温度与传感器数据）
  → 放大器
  → 热泵（Heat Pump）
  → 皮肤接触
```

***

### 3.3 力反馈接口（Force Feedback Interfaces）

与触觉反馈不同，力反馈接口提供**实际反馈力**（而非仅触感）。

#### 力反馈摇杆（Force Feedback Joystick）

*   曾是最廉价、最广泛使用的力反馈设备
*   **WingMan Force 3D（Logitech）**：两个执行器通过机构在摇杆原点产生旋转力矩

***

#### PHANToM 机械臂

*   适用于需要 **3–6 自由度（DoFs）** 及高灵活性的模拟场景

*   6 个 DoF 中：

    *   **3 个主动（Active）**：提供平移方向力反馈
    *   **3 个被动（Passive）**：仅提供朝向，不产生力矩

| 型号              | 峰值力   | 持续力（不过热） | 主动力矩         |
| --------------- | ----- | -------- | ------------ |
| PHANToM Desktop | 6.4 N | 1.7 N    | —            |
| PHANToM 1.5/6.0 | 8.5 N | —        | 170–515 mN·m |


***

#### Haptic Master 机械臂

*   **3DoF 圆柱形机器人**：可绕底座旋转、上下移动、径向伸缩
*   最大输出力：**250 N**，刚度：**5×10⁴ N/m**（极高）
*   **优点**：可模拟坚硬、不可移动的物体
*   **缺点**：惯性较大、价格昂贵

***

#### CyberGrasp 手套

*   需要对各个手指施加独立力时使用
*   将 CyberGlove 与**力控制单元、电气执行器、机械外骨骼**结合
*   最大力：**16 N**
*   重量：**539 g**（长时间使用易产生疲劳）

***

#### CyberForce

*   CyberGrasp 的附加装置，支持模拟**物体重量和惯性**
*   由连接桌面的机械臂 + CyberGrasp 组成
*   腕部最大力：**8.8 N**
*   **不产生力矩**
