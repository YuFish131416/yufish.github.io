## 一、3D 追踪基础（Basics of 3D Tracking）

### 1.1 追踪器的定义

> **Tracker（追踪器）**：实时测量物体**位置（Position）和/或方向（Orientation）**的设备。

*   VR 中常见追踪目标：头部、双手、四肢
*   追踪到的位置与方向，相对于**参考坐标系（World Coordinate System）**来表示

***

### 1.2 自由度（Degrees of Freedom, DOFs）

自由度表示追踪系统可独立测量的运动（或维度）数量。

| 自由度   | 示例设备                    |
| ----- | ----------------------- |
| 2 DOF | 2D 鼠标（仅 x、y 平面位移）       |
| 3 DOF | 加速度计（追踪位置）或陀螺仪（追踪朝向）    |
| 6 DOF | 大多数智能手机（加速度计 + 陀螺仪同时使用） |


> **6 DOF = 3 轴位置（x, y, z）+ 3 轴旋转（Roll, Pitch, Yaw）**，是 VR 中最常见需求。

***

### 1.3 坐标变换矩阵（Coordinate Matrix）

将三维物体点变换到二维图像平面的公式：

```
v2D = Mproj × Mview × Mworld × v3D
```

| 矩阵       | 名称                      | 作用                 |
| -------- | ----------------------- | ------------------ |
| `Mworld` | 模型矩阵（Model Matrix）      | 将点从模型局部坐标系 → 世界坐标系 |
| `Mview`  | 视图矩阵（View Matrix）       | 将点从世界坐标系 → 用户视点坐标系 |
| `Mproj`  | 投影矩阵（Projection Matrix） | 将点从视点坐标系 → 2D 图像平面 |


***

### 1.4 追踪位置与方向

若想追踪用户视点（即头部位置和朝向），将追踪器固定在头部，视图矩阵为：

```
Mview = R(−θroll, −θpitch, −θyaw) × T(−eye)
```

*   **T(−eye)**：平移矩阵，表示眼点（Eye Point）相对参考坐标系的位置![\<img alt="" data-attachment-key="MH8LWIBP" width="439" height="399" src="attachments/MH8LWIBP.png" ztype="zimage"> | 439](attachments/MH8LWIBP.png)
*   **R(−θroll, −θpitch, −θyaw)**：旋转矩阵，由三轴旋转合成：

<!---->

```
R = Rroll(−θroll) × Rpitch(−θpitch) × Ryaw(−θyaw)
```

> 三个旋转轴：**Roll（滚转）、Pitch（俯仰）、Yaw（偏航）**

### 1.5 追踪器性能参数（Tracker Performance）

> ⚠️ **高频考点**：需掌握四个参数的定义及区别

| 参数               | 定义                    | 关键特点                                                       |
| ---------------- | --------------------- | ---------------------------------------------------------- |
| **Accuracy（精度）** | 追踪器测量值与物体真实位置的吻合程度    | 非常数；随距参考坐标系原点的距离增大而下降；可接受精度的范围定义了**工作范围（Operating Range）** |
| **Jitter（抖动）**   | 追踪目标静止时，追踪输出值仍发生的随机波动 | 围绕真值随机浮动；受环境条件影响；在工作空间不同位置可能不同                             |
| **Drift（漂移）**    | 随时间累积的系统性误差增长         | 因误差累积导致，需定期用辅助追踪器**重置误差**                                  |
| **Latency（延迟）**  | 物体运动发生到追踪器输出信号之间的时间延迟 | 延迟过大会导致**仿真病（Simulation Sickness）**：头痛、恶心、眩晕               |


**降低延迟的方法：**

*   使用更快的通信线路
*   减少每个处理步骤的延迟
*   同步追踪测量、通信、渲染和显示循环（降低整体更新率抖动）

***

### 1.6 追踪器更新率（Tracker Update Rate）

*   每秒报告的测量次数，通常 **≥ 30 次/秒**
*   更新率越高，仿真的动态响应越好
*   ⚠️ 若一个追踪系统需支持**多个传感器**（追踪多个身体部位），采样率需**按比例降低**

***

## 二、3D 追踪器类型（Types of 3D Trackers）

### 2.1 机械追踪器（Mechanical Trackers）

**工作原理：**

*   由**串联或并联运动链（Links + Sensorized Joints）**构成
*   系统预知运动链的结构；通过关节传感器得到各部分相对位置和方向
*   将一端固定在地面/桌面，另一端即可实时计算目标的 3D 位置

**典型案例：Gypsy Motion Capture**

*   传感器外骨骼套装，测量 **15 个关节** 的位置
*   使用 **42 个精密电位计（Potentiometers）**，精度达 0.08°
*   通过脚部微动开关检测步行时脚的着地状态，计算双脚相对位置
*   Gypsy 2.5 版本加入陀螺仪追踪髋部旋转
*   重量：5\~7 kg

**优点：**

*   精度高（Accurate）
*   延迟小（Small Latency）
*   无视线遮挡限制（No Line-of-Sight Restriction）
*   长时间记录漂移小（Small Drift）
*   追踪工作空间无限（相对基座的运动）

**缺点：**

*   运动范围受机械臂长度**限制**
*   质量大 → 机械振荡 → 惯性干扰
*   沉重 → 用户疲劳，降低沉浸感
*   机械结构**干扰（Intrusive）**用户的自然运动

> 非接触式追踪器（磁场、超声波、光学、惯性）不存在运动干扰问题。

***

### 2.2 磁场追踪器（Magnetic Trackers）

**工作原理：**

*   固定**发射器（Transmitter）**：3 个相互正交线圈绕在铁磁立方体上，顺序激励产生 3 个正交磁场
*   **接收器（Receiver）**附着在被追踪物体上：由 3 个正交线圈（AC）或磁力计（DC）组成
*   接收器从 9 个电压值中解算出位置 (x, y, z) 和方向 (θx, θy, θz)
*   **时分复用（Time Multiplexing）**允许多个传感器同时连接

#### AC 磁场追踪器

| 产品                     | 特点                                                         |
| ---------------------- | ---------------------------------------------------------- |
| Polhemus Isotrak（1991） | 首款 AC 磁场追踪器；延迟约 30ms；抖动约 1°（导致虚拟手抖动）                       |
| Polhemus Fastrak       | DSP 架构；单传感器时 120 次/秒；双传感器时降至 60 次/秒；工作范围 75cm（长距离可扩展至 3 倍） |


**AC 追踪器的核心问题：涡流（Eddy Currents）**

*   变化磁场在周围导体中感应出涡流
*   涡流产生反向磁场，**干扰原始磁场** → 接收器收到失真磁场 → 精度下降
*   感应电压公式：`Vr ∝ N × dB/dt`（N = 线圈匝数，dB/dt = 磁场变化率）

#### DC 磁场追踪器

**解决方案：** 用脉冲 DC 磁场替代 AC 磁场，在激励间隔中让涡流自然消散

**时序周期：**

*   T0–T1：所有磁场关闭（测量地球磁场本底值）
*   T1–T2：X 轴磁场开启
*   T2–T3：Y 轴磁场开启
*   T3–T4：Z 轴磁场开启
*   最终从各周期读数中减去地球磁场的影响

| 产品                             | 特点                                                                             |
| ------------------------------ | ------------------------------------------------------------------------------ |
| Ascension Flock of Birds（1998） | 分布式计算架构；每个接收器独立处理器；30 个传感器仍保持 144 次/秒更新率                                       |
| Ascension MotionStar（2001）     | 无线系统；双发射器（ERT1+ERT2）交替工作；10~20 个接收器固定在身体上；背包（1.7kg）含电池+处理器+无线调制解调器；可追踪最多 5 名用户 |


**DC 追踪器的问题：**

*   **环境误差**：磁场强度随距离的立方衰减，误差公式：`Error = K × d⁴`

*   **金属干扰**：

    *   DC 追踪器**不受**非铁磁金属（黄铜、铝、不锈钢）影响
    *   AC/DC 追踪器**都受**铁磁金属（低碳钢、铁氧体）影响（其会被磁化进而干扰磁场）
    *   铜（高导电率）也会让涡流持续时间超过 DC 的等待间隔 → 影响 DC 追踪器

**磁场追踪器总结：**

| 优点            | 缺点         |
| ------------- | ---------- |
| 精度较好（不及机械追踪器） | 受金属物体/表面干扰 |
| 传感器轻便         |            |
| 无视线遮挡限制       |            |

### 2.3 超声波追踪器（Ultrasonic Trackers）

**工作原理：**

*   **发射器**：3 个超声扬声器安装在约 30cm 间距的固定三角架上
*   **接收器**：3 个超声麦克风安装在较小的三角框架上
*   利用已知声速，通过**飞行时间（Time of Flight）**计算距离
*   每个扬声器依次激活，测量到 3 个麦克风的距离，共 9 段距离
*   由此解算接收器平面的**位置和方向**

**关键参数：**

*   更新率：约 **50 次/秒**（远低于磁场追踪器）
*   低更新率原因：需等待 **5\~100ms** 让前次回声消散后才能开始新一次测量
*   工作范围：约 **1.52m**（受超声波空气衰减限制）
*   需要**直接视线（Direct Line of Sight）**
*   需消除背景噪声

**扩大追踪范围：** 多个发射器空间复用（同时只能开启一个），追踪区域需重叠以实现无缝追踪

**超声波追踪器总结：**

| 优点      | 缺点                |
| ------- | ----------------- |
| 不受金属干扰  | 需直接视线             |
| 体积小、重量轻 | 精度受声速（温度、气压、湿度）影响 |
| 相对便宜    | 易受环境噪声影响（如墙面反射）   |


***

### 2.4 光学追踪器（Optical Trackers）

**特点：**

*   非接触式位置测量
*   高更新率，小延迟
*   需要**直接视线**

#### 方式一：Outside Looking In（外部观察内部）

*   传感器（摄像机）固定在**外部**；标记物（Markers/光学反射器）附着在**被追踪对象**上

*   典型系统：**MotionAnalysis 光学运动捕捉系统**

    *   工作空间周围安装红外光源
    *   被追踪对象上附红外反射标记
    *   多台红外摄像机（如 8 台）同时捕捉标记反射
    *   仅需在至少 3\~4 台摄像机中识别同一标记，即可重建其 3D 位置

**运动捕捉流程（5步）（Optical Motion Capture System Pipeline）：**

| 步骤 | 名称                                     | 内容                            |
| -- | -------------------------------------- | ----------------------------- |
| 1  | **Calibration（标定）**                    | 使用棋盘格或标定架，确定每台摄像机在参考坐标系中的位置   |
| 2  | **Motion Capture（运动捕捉）**               | 演员执行待捕捉动作                     |
| 3  | **3D Position Reconstruction（3D位置重建）** | 每个标记须被至少 2 台摄像机同时看到才能重建 3D 位置 |
| 4  | **Fitting the Skeleton（骨骼拟合）**         | 将标记关联到预定义骨骼，提升精度              |
| 5  | **Post-Processing（后处理）**               | 去噪、调整速度、动作编辑等                 |


**追踪精度退化情形：**

*   标记间距越小 → 计算精度下降
*   用户距摄像机越远 → 计算误差增大
*   相邻摄像机间距越大 → 遮挡导致视角差异增大（但减少间距需更多摄像机）

**优/缺点：**

| 优点            | 缺点                                    |
| ------------- | ------------------------------------- |
| 高精度（大量校准算法支持） | **遮挡问题**：身体部位互相遮挡时无法恢复（若有3D模型可辅助减少误差） |


#### 方式二：Inside Looking Out（内部观察外部）

*   传感器（摄像机）**附在被追踪对象上**；标记物固定在**墙壁**上

*   对**方向变化**的敏感性最大

*   工作空间理论上可**无限扩展**

*   典型系统：**HiBall Tracker（3rd Tech）**

    *   两个 HiBall 感应单元 + LED 信标阵列 + 天花板接口 + 主机
    *   6 个窄视角（6°）镜头排列在半球的 6 个扇区
    *   侧效应光电二极管（LEPD）检测 LED 信标图像的 x-y 坐标
    *   信标阵列：6 排×8 LED，共 48 个 LED，覆盖约 8 ft²
    *   多个阵列可将追踪范围从 8×8 ft 扩展到 40×40 ft
    *   更新率：**2000 次/秒**（极高）

### 2.5 惯性传感器（Inertial Sensors）

三种传感器类型：

| 传感器                     | 测量量   | 单位   | 特性              |
| ----------------------- | ----- | ---- | --------------- |
| **陀螺仪（Gyroscope）**      | 角速度   | °/s  | 短期精准；长期存在**漂移** |
| **加速度计（Accelerometer）** | 线性加速度 | m/s² | 长期稳定；短期**噪声大**  |
| **磁力计（Magnetometer）**   | 地磁场强度 | μT   | 受附近金属影响；通用性较低   |


**陀螺仪原理（科里奥利效应）：**

*   传感器形如音叉，持续振动
*   旋转时科里奥利力使两元件在垂直方向也振动
*   科里奥利振动 → 转换为与角速度成正比的电压 → 3 组正交组合得到 3D 方向
*   通过**积分角速度** → 计算 3D 方向
*   受温度影响，长期存在漂移

**加速度计原理：**

*   3 轴加速度计与 3 轴陀螺仪共轴安装
*   **双重积分加速度** → 3D 位置（需减去重力和旋转效应）
*   短期噪声大，但长期不漂移 → **与陀螺仪互补**

**IMU（惯性测量单元）示例：InvenSense MPU-9250**

*   9 DOF：3轴陀螺仪 + 3轴加速度计 + 3轴磁力计
*   包含 9 个 16-bit ADC

**惯性追踪器总结：**

| 优点              | 缺点                       |
| --------------- | ------------------------ |
| 追踪范围无限          | 误差快速累积（漂移）               |
| 无视线遮挡限制         | 陀螺仪偏置 → 方向误差随时间**线性**增长  |
| 低抖动（Low Jitter） | 加速度计偏置 → 位置误差随时间**平方**增长 |
|                 | 商业设备：约 2s 内产生 40mm 误差    |
|                 | **解决方案**：定期用另一种追踪器重置误差   |


***

### 2.6 3D 导航/操控设备（3D Navigation/Manipulation Devices）

*   **Navigation（导航）**：在虚拟环境中交互式改变视角，用于探索
*   **Manipulation（操控）**：选择并修改对象

#### Logitech Magellan

*   传感化圆柱体，测量用户施加的 **3 个力 + 3 个力矩**（共 6 DOF）
*   内部 6 个 LED + 6 个光传感器，通过外壳形变测量力/力矩
*   **缺点**：传感器耦合（Sensor Coupling）——用户想平移时往往同时产生旋转；用户不够友好

#### MicroScribe 3D（3D 探针）

*   6 个旋转关节 → 探针有 6 DOF（可同时定位 + 定向）
*   配有配重（靠近底座），减轻用户疲劳
*   用于**数字化真实物体（Digitize Objects）**
*   探针位置通过**正向运动学（Direct Kinematics）**计算
*   脚踏板二进制开关用于选择/取消虚拟对象或标记点
*   可达 1000+ 次采样/秒
*   **误差从底座向末端累积** → 需校准

## 三、手势设备（Gesture Devices）

> 实时测量用户手指位置，支持基于手势的虚拟环境交互。

### 3.1 各手套设备对比

| 设备                  | 传感原理                                              | 分辨率/特点                       |
| ------------------- | ------------------------------------------------- | ---------------------------- |
| **Pinch Gloves**    | 导电纤维片（指尖、手指背、手心）；检测电接触建立/断开                       | 仅检测是否有接触，无连续角度值              |
| **5DT Data Gloves** | 每根手指一个光纤传感器；LED + 光电晶体管检测弯曲时漏光量                   | 8-bit A/D → 256 级分辨率；需归一化处理  |
| **Didjigloves**     | 每根手指电容弯曲传感器；两层导电聚合物重叠面积 ∝ 弯曲角度 ∝ 电容量              | 10 个电容传感器                    |
| **CyberGloves**     | 弹性尼龙手套上的应变片（Strain Gauges）；差分电阻变化 → 差分电压 → A/D 转换 | 18 或 22 个传感器；测量屈曲、展收、腕部俯仰/偏航 |


### 3.2 5DT Data Gloves 详解

**归一化公式：**

```
out = (rawval − rawmin) / (rawmax − rawmin) × 256
```

*   rawmax − rawmin：特定传感器对特定用户的**动态范围**
*   校准方式：用户多次弯曲手指，系统自动更新动态范围
*   系统生成**查找表（Look-up Table）**：将传感器输出映射到姿态

### 3.3 CyberGloves 详解

**传感器类型：**

*   **矩形应变片**：测量屈曲角度（Flexion）
*   **U 形应变片**：测量展收角度（Adduction-Abduction）

**工作机制：**

*   手指运动时，一个应变片受**压缩**，另一个受**拉伸**
*   电阻变化 → 差分电压 → 放大 → A/D 转换

**优势：** 传感器数量多、编程支持好、可扩展为更复杂的触觉手套

## 考点速览表

| 考核知识点            | 核心内容                                          |
| ---------------- | --------------------------------------------- |
