## 一、特征不变性（Lec05: Feature Invariance）

### 1.1 不变性 vs 等变性（Invariance vs Equivariance）

这是理解局部特征的基础概念：

**不变性（Invariance）**：图像发生变换后，**特征位置不发生变化**。

> 例：对亮度整体偏移的不变性——图像变亮后角点仍然在同一位置。

**等变性（Equivariance / Covariance）**：图像发生变换后，**特征位置发生相应的同等变换**。

> 例：图像旋转 90°，特征点也随之旋转 90°，保持对应关系。

> 注意："不变性"在很多文献中泛指这两者。

***

### 1.2 Harris 检测器的不变性分析

| 变换类型                      | Harris 的性质 | 说明                                 |
| ------------------------- | ---------- | ---------------------------------- |
| **平移（Translation）**       | ✅ 等变       | 导数与窗口函数均等变，角点位置随之平移                |
| **旋转（Rotation）**          | ✅ 等变       | 二阶矩椭圆整体旋转，但特征值（形状）不变               |
| **仿射亮度变化** $I \to aI + b$ | ⚠️ 部分不变    | 对偏移 $b$ 不变；对尺度 $a$ 部分不变（阈值会影响检测结果） |
| **尺度变化（Scaling）**         | ❌ 不具备      | 大尺度下角点可能被识别为边缘，既不不变也不等变            |


***

### 1.3 尺度不变检测（Scale-Invariant Detection）

**核心问题**：同一角点在不同尺度下表现不同，Harris 无法处理尺度变化。

**解决思路**：在**位置和尺度空间**中同时寻找响应的局部极大值（找"特征尺度"）。

**实现方法**：

1.  构建**高斯金字塔**（在不同尺度的图像上计算响应）
2.  找到在某一尺度下响应最大的点，该尺度即为**特征尺度**

***

### 1.4 LoG 斑点检测器⭐重点

**LoG（Laplacian of Gaussian）**：高斯的拉普拉斯算子

\$\$\text{LoG} = \nabla^2 G\_\sigma = \frac{\partial^2 G}{\partial x^2} + \frac{\partial^2 G}{\partial y^2}\$\$

**核心作用**：斑点（Blob）检测器，对圆形区域有强响应。

**特征尺度的定义**：LoG 响应取得峰值时对应的 \$\sigma\$ 即为该斑点的**特征尺度（Characteristic Scale）**。

**DoG（Difference of Gaussians）**： \$\$\text{DoG} = G\_{\sigma\_1} - G\_{\sigma\_2} \approx \text{LoG}\$\$

两个不同 \$\sigma\$ 的高斯之差近似等于 LoG，计算效率更高（SIFT 使用的方法）。

**检测流程**：

1.  在多尺度（多层 \$\sigma\$）下计算 LoG（或 DoG）响应图
2.  在**三维（x, y, scale）空间**中寻找局部极大/极小值
3.  输出每个斑点的位置 \$(x, y)\$ 和特征尺度 \$s\$

| 比较项  | Harris     | LoG/DoG  |
| ---- | ---------- | -------- |
| 检测类型 | 角点（corner） | 斑点（blob） |
| 尺度不变 | ❌          | ✅        |
| 旋转等变 | ✅          | ✅        |

## 二、特征描述子与特征匹配（Lec06: Descriptors & Matching）

### 2.1 局部特征的三步骤框架

```
Detection（检测）→ Description（描述）→ Matching（匹配）
```

### 2.2 描述子的两大要求

**不变性（Invariance）**：图像变换（旋转、尺度、亮度）后描述子不变。

**可区分性（Discriminability）**：每个点的描述子高度唯一，能准确区分不同点。

***

### 2.3 MOPS 描述子（Multiscale Oriented Patches）

**步骤**：

1.  检测点周围取 40×40 窗口
2.  预滤波后缩放到 8×8（1/5 大小）
3.  旋转到主方向（水平化）
4.  强度归一化（减均值，除标准差）→ 提升光照不变性

***

### 2.4 SIFT 描述子（Scale-Invariant Feature Transform）⭐重点

**来源**：David Lowe, UBC

**主要步骤**：

1.  取检测点周围 **16×16** 像素窗口
2.  计算每个像素的**梯度方向**
3.  剔除弱梯度像素（幅值阈值）
4.  将窗口划分为 **4×4** 的网格
5.  每个格子内统计 **8 个方向**的梯度直方图
6.  按主方向旋转，实现旋转不变性
7.  拼接所有格子的直方图 → **128维向量**（4×4×8）

**SIFT 的性质**：

*   对约 **60°** 的平面外旋转具有鲁棒性
*   可处理从白天到夜晚的光照变化
*   单张中等图像 < 1 秒，难以实时

***

### 2.5 其他描述子

| 描述子       | 全称                                  | 特点                 |
| --------- | ----------------------------------- | ------------------ |
| **HOG**   | Histogram of Oriented Gradients     | 行人检测；局部区域梯度直方图     |
| **ORB**   | Oriented FAST and Rotated BRIEF     | 二进制描述子，实时，设备端 SLAM |
| **FREAK** | Fast Retina Keypoint                | 受视网膜启发，可实时         |
| **LIFT**  | Learned Invariant Feature Transform | 深度学习，端到端学习         |


**HOG 详细步骤**：

1.  计算每个像素的梯度（幅值 + 方向）
2.  将图像划分为 8×8 的 cell，构建 9 个 bin 的直方图（0–180°）
3.  将 cell 分为重叠 block（2×2 cells），做 L2 归一化

**ORB（rBRIEF + oFAST）**：

*   用 FAST 检测角点 + Harris 度量排序
*   BRIEF 二进制描述子（256位），旋转不变

***

### 2.6 特征匹配⭐重点

#### ① L2 距离（简单方法）

\$\$d(f\_1, f\_2) = | f\_1 - f\_2 |\$\$

*   缺点：可能给模糊匹配赋予较小距离值。

#### ② 比率距离（Ratio Distance）⭐推荐方法

\$\$\text{ratio} = \frac{| f\_1 - f\_2 |}{| f\_1 - f\_2' |}\$\$

*   \$f\_2\$：最佳匹配，\$f\_2'\$：第二佳匹配
*   **效果**：正确匹配比值小（远优于第二佳）；错误匹配比值接近1（两个候选差不多）
*   **不改变最佳匹配**，但改变距离值与排序，便于阈值筛选

***

### 2.7 匹配性能评估

| 指标                       | 定义                                       |
| ------------------------ | ---------------------------------------- |
| 真阳性（TP, True Positives）  | 通过阈值 且 正确的匹配                             |
| 假阳性（FP, False Positives） | 通过阈值 但 错误的匹配                             |
| 真阳性率（TPR / Recall）       | $\text{TPR} = \text{TP} / \text{正确匹配总数}$ |
| 假阳性率（FPR）                | $\text{FPR} = \text{FP} / \text{错误匹配总数}$ |


**ROC 曲线（Receiver Operating Characteristic）**：

*   横轴：假阳性率；纵轴：真阳性率
*   通过扫描所有阈值绘制曲线
*   **AUC（Area Under Curve）**：越接近1越好，综合评价描述子性能

**示例**：

> 1000个匹配（800正确，200错误），阈值保留 600正确 + 100错误 TPR = 600/800 = 0.75，FPR = 100/200 = 0.50

## 三、图像变换与图像变形（Lec07: Transformations & Warping）

### 3.1 图像操作的两类方式

| 类型                  | 定义                   | 公式               |
| ------------------- | -------------------- | ---------------- |
| **图像滤波（Filtering）** | 改变图像的**值域（range）**   | $g(x) = h(f(x))$ |
| **图像变形（Warping）**   | 改变图像的**定义域（domain）** | $g(x) = f(h(x))$ |


***

### 3.2 线性变换（2×2 矩阵）

\$\$\begin{bmatrix}x'\y'\end{bmatrix} = \begin{bmatrix}a\&b\c\&d\end{bmatrix}\begin{bmatrix}x\y\end{bmatrix}\$\$

**可表示的变换**：缩放（Scale）、旋转（Rotation）、剪切（Shear）、镜像（Mirror）

**不可表示的变换**：**平移**（Translation）——平移不是线性变换！

**线性变换的性质**：

*   原点映射到原点
*   直线映射到直线
*   平行线保持平行
*   比例关系保持不变

常见矩阵：

| 变换     | 矩阵                                                                               |
| ------ | -------------------------------------------------------------------------------- |
| 旋转 θ   | $\begin{bmatrix}\cos\theta & -\sin\theta \ \sin\theta & \cos\theta\end{bmatrix}$ |
| 均匀缩放 s | $\begin{bmatrix}s & 0 \ 0 & s\end{bmatrix}$                                      |
| Y轴镜像   | $\begin{bmatrix}-1 & 0 \ 0 & 1\end{bmatrix}$                                     |


***

### 3.3 齐次坐标（Homogeneous Coordinates）⭐重点

**为什么需要齐次坐标**：为了用统一的矩阵乘法表示**平移**。

**表示方式**：2D 点 \$(x, y)\$ → 齐次坐标 \$(x, y, 1)\$，即在末尾追加 \$w=1\$。

**从齐次坐标恢复**：\$(x, y, w) \to (x/w, y/w)\$

**平移的齐次表示（3×3矩阵）**： \$\$\begin{bmatrix}x'\y'\1\end{bmatrix} = \begin{bmatrix}1&0\&t\_x\0&1\&t\_y\0&0&1\end{bmatrix}\begin{bmatrix}x\y\1\end{bmatrix}\$\$

***

### 3.4 仿射变换（Affine Transformation）⭐重点

**定义**：最后一行为 $\[0\ 0\ 1]\$ 的 3×3 矩阵所表示的变换。

\$\$\begin{bmatrix}x'\y'\1\end{bmatrix} = \begin{bmatrix}a\_{11}\&a\_{12}\&t\_x\a\_{21}\&a\_{22}\&t\_y\0&0&1\end{bmatrix}\begin{bmatrix}x\y\1\end{bmatrix}\$\$

**仿射变换 = 线性变换 + 平移**，包含：平移、旋转、缩放、剪切（6个自由度）。

**仿射变换的性质**：

*   原点**不一定**映射到原点
*   直线映射到直线
*   **平行线保持平行** ← 关键性质
*   比例关系保持不变

***

### 3.5 投影变换（Projective Transformation）/ 单应性（Homography）⭐重点

**定义**：3×3 矩阵，**最后一行不再为** $\[0\ 0\ 1]\$，因此分母可以不为1。

\$\$\begin{bmatrix}wx'\wy'\w\end{bmatrix} = \begin{bmatrix}h\_{00}\&h\_{01}\&h\_{02}\h\_{10}\&h\_{11}\&h\_{12}\h\_{20}\&h\_{21}\&h\_{22}\end{bmatrix}\begin{bmatrix}x\y\1\end{bmatrix}\$\$

\$\$x' = \frac{h\_{00}x + h\_{01}y + h\_{02}}{h\_{20}x + h\_{21}y + h\_{22}}, \quad y' = \frac{h\_{10}x + h\_{11}y + h\_{12}}{h\_{20}x + h\_{21}y + h\_{22}}\$\$

**关键性质**：

*   \$H\$ 和 \$2H\$ 等价（只定义到尺度因子，有 **8 个自由度**）
*   直线映射到直线
*   **平行线不一定保持平行** ← 关键区别于仿射
*   比例关系不保持
*   可表示透视（perspective）效果

**变换层次关系（嵌套）**：

```
线性变换（2×2）
    ⊂ 仿射变换（3×3，最后行 [0 0 1]）
        ⊂ 投影变换（单应性）（3×3，任意最后行）
```

***

### 3.6 图像变形的两种实现方式

#### 正向变形（Forward Warping）

*   将每个源像素 \$(x,y)\$ 映射到目标位置 \$(x',y')\$
*   问题：像素可能落在非整数位置，可能产生**空洞（holes）**

#### 逆向变形（Inverse Warping）⭐推荐

*   对目标图像每个像素 \$(x',y')\$，反查变换 \$T^{-1}(x',y')\$ 得到源坐标
*   通过**插值**（最近邻/双线性/双三次）获取源像素值
*   不会产生空洞，实践中更常用

***

## 四、图像对齐与最小二乘法（Lec08: Image Alignment & Least Squares）

### 4.1 图像对齐的总体流程

```
Step 1: 提取两幅图像 A 和 B 的特征点
Step 2: 匹配 A 和 B 之间的特征点
Step 3: 用最小二乘法计算从 A 到 B 的变换 T
```

### 4.2 最小二乘法（Least Squares）⭐重点

**问题设置**：给定 \$n\$ 个点对 \$(p\_i, p\_i')\$，求最优变换 \$T\$。

**残差（Residual）**： \$\$r\_i = p\_i' - T(p\_i)\$\$

**目标函数（最小化残差平方和）**： \$\$\min\_T \sum\_{i=1}^{n} r\_i^2 = \min\_T | A\mathbf{t} - \mathbf{b} |^2\$\$

**正则方程（Normal Equations）**： \$\$A^T A \mathbf{t} = A^T \mathbf{b}\$\$ \$\$\mathbf{t} = (A^T A)^{-1} A^T \mathbf{b}\$\$

**特例**：对平移变换，最小二乘解 = **所有位移向量的均值**。

***

### 4.3 各变换的最小二乘求解

#### 平移变换（2 个未知数）

*   每个点对提供 **2 个方程**
*   最少需要 **1 对点**（理论上）；实际用多对点取均值

#### 仿射变换（6 个未知数）

\$\$\begin{bmatrix}x'\y'\end{bmatrix} = \begin{bmatrix}a\&b\&c\d\&e\&f\end{bmatrix}\begin{bmatrix}x\y\1\end{bmatrix}\$\$

*   每个点对提供 **2 个方程**
*   最少需要 **3 对点**；实际超定用最小二乘

矩阵形式（\$2n \times 6\$）：

\$\$\mathbf{Ap} = \mathbf{b}, \quad A \in \mathbb{R}^{2n \times 6}, \quad \mathbf{p} \in \mathbb{R}^6\$\$

#### 单应性（Homography，8 个自由度）

*   每个点对提供 **2 个方程**
*   **最少需要 4 对点**

**求解方法**：

1.  将约束改写为线性方程组 \$A\mathbf{h} = \mathbf{0}\$（\$2n \times 9\$ 矩阵）
2.  由于 \$H\$ 只定义到尺度，求单位向量解
3.  **解 = \$A^T A\$（或 \$A\$）最小特征值对应的特征向量**（SVD 分解）

***

### 4.4 离群点（Outliers）问题

**问题**：特征匹配中存在错误匹配（outliers），最小二乘对 outliers 非常敏感，会拉偏结果。

**需要更鲁棒的方法** → RANSAC（见下一节）。

***

## 五、RANSAC 鲁棒估计（Lec09）

### 5.1 为什么需要 RANSAC？

**最小二乘的缺陷**：一旦有 outlier（错误匹配），整个拟合结果都会偏移。

**RANSAC 的核心思想**：

> "所有内点（inliers）对变换达成一致；每个外点（outlier）以自己的方式错误。"

***

### 5.2 RANSAC 算法⭐重点

**全称**：Random Sample Consensus（随机样本一致性）

**算法流程**：

```
重复 N 次：
    Step 1: 随机选取 s 个点对（最小样本集）
    Step 2: 用这 s 个点对拟合一个模型（如：计算单应性）
    Step 3: 计算所有其他点对在该模型下的残差
    Step 4: 统计残差 < 阈值 ε 的点对数量（inliers）
最终：选取 inlier 数量最多的模型
后处理：用所有 inliers 重新用最小二乘拟合最终模型
```

**参数说明**：

*   \$s\$：最小样本集大小（由模型自由度决定）
*   \$\varepsilon\$（epsilon）：inlier 判定阈值（通常设为噪声标准差的 2\~3 倍）
*   \$N\$：迭代轮数（由 outlier 比例和所需成功概率决定）

***

### 5.3 迭代轮数 N 的计算⭐重点

**推导**：

*   设 outlier 比例为 \$e\$（\$0 < e < 1\$）
*   每次随机取 \$s\$ 个点，全部为 inlier 的概率：\$(1-e)^s\$
*   \$N\$ 次中至少一次全是 inlier 的概率 \$\geq p\$：

\$\$1 - (1-(1-e)^s)^N \geq p\$\$

\$\$N \geq \frac{\log(1-p)}{\log(1-(1-e)^s)}\$\$

**查表（\$p = 0.99\$）**：

| $s$（样本数） | outlier 20% | outlier 30% | outlier 40% | outlier 50% |
| -------- | ----------- | ----------- | ----------- | ----------- |
| 2        | 5           | 7           | 11          | 17          |
| 4        | 9           | 17          | 34          | 72          |
| 8        | 26          | 78          | 272         | 1177        |


> 关键结论：\$s\$ 越大、outlier 比例越高，所需轮数 \$N\$ 急剧增加。

***

### 5.4 各模型的最小样本数 \$s\$

| 模型                  | 自由度   | 最小样本数 $s$ |
| ------------------- | ----- | --------- |
| 平移                  | 2     | 1 对点      |
| 仿射变换                | 6     | 3 对点      |
| **单应性（Homography）** | **8** | **4 对点**  |


***

### 5.5 RANSAC 的优缺点

| 优点          | 缺点                        |
| ----------- | ------------------------- |
| 简单通用，适用多种问题 | 参数（$\varepsilon$、$N$）需要调整 |
| 实践中效果好      | outlier 比例 > 50% 时失效      |
| 不需要所有点都正确   | 极端情况下迭代次数过多               |


***

### 5.6 完整图像拼接（Panorama）流程

```
Step 1: 检测特征点（Harris / DoG）
Step 2: 计算描述子（SIFT）
Step 3: 特征匹配（最近邻 + 比率距离）
Step 4: RANSAC 鲁棒估计单应性 H
Step 5: 逆向变形（Inverse Warping）+ 插值
Step 6: 合并图像
```

| 等变 vs 不变   | 等变：变换后特征位置随之变换；不变：位置不变                                     |

| LoG 特征尺度   | LoG 响应峰值处的 $\sigma$ 即为特征尺度                                 |
| DoG        | $G\_{\sigma\_1} - G\_{\sigma\_2} \approx \text{LoG}$（计算更快） |
| SIFT 维度    | $4 \times 4 \times 8 = \mathbf{128}$ 维                     |
| 比率距离       | $\frac{\|f\_1-f\_2\|}{\|f\_1-f\_2'\|}$，正确匹配比值小             |
| 仿射变换自由度    | 6（平移2 + 线性4）                                               |
| 单应性自由度     | 8（3×3矩阵，仅定义到尺度）                                            |
| 求单应性所需点数   | **最少 4 对点**                                                |
| 单应性求解      | SVD：$A\mathbf{h}=\mathbf{0}$ 的最小特征值对应特征向量                  |
| RANSAC 迭代数 | $N \geq \frac{\log(1-p)}{\log(1-(1-e)^s)}$                 |
| 仿射 vs 射影   | 仿射保平行线；射影（单应性）不保平行线                                        |
| 逆向变形       | 推荐实现方式，配合插值，无空洞                                            |

## 八、易混淆概念辨析

**仿射变换 vs 单应性**：

*   仿射：最后行固定为 $\[0\ 0\ 1]\$，6自由度，**保平行线**
*   单应性：最后行任意，8自由度，**不保平行线**，可表示透视变形

**LoG vs DoG**：

*   LoG：精确的高斯拉普拉斯；DoG：两个高斯之差近似 LoG，计算高效（SIFT 使用 DoG）

**正向变形 vs 逆向变形**：

*   正向：源→目标，可能有空洞；逆向：目标→源，用插值填充，无空洞，**实践中用逆向**

**RANSAC inlier 阈值 vs 迭代轮数**：

*   阈值控制"什么算同意"（与噪声水平相关）
*   迭代轮数控制"找到正确解的概率"（与 outlier 比例相关）

**比率距离的作用**：

*   不改变最佳匹配对象
*   但改变距离度量，使"歧义匹配"（两个候选差不多）的比值接近1，便于用阈值过滤
