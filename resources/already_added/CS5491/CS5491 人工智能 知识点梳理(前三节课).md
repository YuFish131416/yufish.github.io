## **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">二、搜索空间表示（Search Space Representation）</span></span>**

### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">核心概念详解：</span></span>**

#### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">1. 状态空间图 vs 搜索树</span></span>**

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">状态空间图（State Space Graph）</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>

    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">节点=状态，边=状态间的转移</span></span>
    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">每个状态只出现一次</span></span>
    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">能有效处理重复状态</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">搜索树（Search Tree）</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>

    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">节点=从初始状态到达该状态的路径</span></span>
    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">同一状态可能在不同路径中多次出现</span></span>
    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">不处理重复状态时可能无限大</span></span>

#### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">2. 通用搜索算法框架</span></span>**

```
function TREE-SEARCH(problem, strategy):
    frontier = {initial state}
    while frontier not empty:
        node = select node from frontier according to strategy
        if node is goal: return solution
        expand node and add children to frontier
```

function GRAPH-SEARCH(problem, strategy):
    frontier = {initial state}
    explored = {}
    while frontier not empty:
        node = select node from frontier according to strategy
        if node is goal: return solution
        add node to explored
        expand node and add children to frontier only if not in frontier or explored

>     2.  <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">它包含了所有可能的状态和所有可能的连接，但通常</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">太大而无法全部画出来</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
> **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">你的“探索记录”——搜索树</span></span>**
>
> *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">把搜索树想象成你在探索上面那张地图时</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">亲手画的探险笔记</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">节点</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：不是你所在的地点，而是 </span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">“你如何到达这里的一整条路径”</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"> 。例如，节点不是“锡比乌(Sibiu)”，而是“</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">从阿拉德(Arad)到锡比乌</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">”这个完整事件。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">边</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：你的</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">下一步行动</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">关键特性</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>
>
>     1.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">同一个状态（地点）可能会在树中多次出现</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。比如，在你的探险笔记里，“阿拉德”这个城市可能会出现很多次，因为你从不同路径又绕了回来。</span></span>
>
>     2.  <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">如果不处理重复状态，这棵树可能会</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">无限生长</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">下去（比如不停地在几个城市间绕圈）。</span></span>
>
> #### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">两者对比</span></span>**
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">状态空间图</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">是客观存在的、完整的</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">世界模型</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">（地图）。它是问题的“真理”。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">搜索树</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">是搜索算法在</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">探索过程中</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">动态构建的、主观的</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">计划与记录</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">（探险笔记）。</span></span>
>
> *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">因为世界太大，我们无法拥有完整地图，也无法记下所有可能的路线。所以搜索算法都是</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">“边探索边画图”</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，只构建它们认为需要的部分。</span></span>
>
> <!---->
>
> ```
> 树搜索 vs 图搜索
> ```
>
> <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">这是两种基本的搜索框架，区别在于是否记录“去过哪里”</span></span>
>
> **树搜索 (莽夫式搜索):**
>
> *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">它只维护一个</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">边界（待探索节点列表）</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
> *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">它</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">不记得自己已经去过哪些状态</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">后果</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：它可能会在状态空间里</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">无限循环</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，比如从A到B，又从B回到A，永远绕不出去。就像在一个迷宫里不做标记，很容易重复走老路。</span></span>
>
> **图搜索 (聪明的搜索，会做标记):**
>
> *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">它维护两个集合：</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">边界</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"> 和 </span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">已探索集合</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">它会记录所有已经扩展过的状态</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
> *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">当扩展一个节点产生新状态时，它会检查这个状态</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">是不是已经在边界里或者已经探索过了</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。如果是，就忽略它。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">后果</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：它</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">永远不会重复探索同一个状态</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，因此不会陷入循环。就像在迷宫里用粉笔做标记，走过的路绝不走第二遍。</span></span>
>
> <!---->
>
> ```
> 如何评判一个搜索算法的好坏
> ```
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">策略</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：决定下一步探索哪个节点。是“广撒网”（BFS）还是“一条路走到黑”（DFS）？</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">四个评价标准</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>
>
>     1.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">完备性</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：这个算法靠谱吗？能给个交代吗？（有解就一定能找到吗？）</span></span>
>
>     2.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">最优性</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：这个算法聪明吗？能找到最好的那条路吗？</span></span>
>
>     3.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">时间</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：这个算法快吗？需要尝试多少种可能？</span></span>
>
>     4.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">空间</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：这个算法吃内存吗？需要记多少东西？</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">三个关键参数</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>
>
>     *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">分支因子(b)</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：平均有多少种下一步可走。就像树杈多不多。</span></span>
>
>     *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">目标深度(d)</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：最快能赢的步骤是多少步。</span></span>
>
>     *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">最大深度(m)</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：最坏情况要走多少步（可能无限循环）。</span></span>
>
> <!---->
>
> ```
> 总结给初学者：
> ```
>
> 1.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">建模</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：把问题转化成</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">状态空间</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">是第一步，也是最难的一步。</span></span>
>
> 2.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">两个核心概念</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>
>
>     *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">状态空间图</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：是客观的</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">世界全貌</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">（地图）。</span></span>
>
>     *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">搜索树</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：是主观的</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">探索过程</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">（探险笔记）。</span></span>
>
> 3.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">两种搜索框架</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>
>
>     *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">树搜索</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：简单，但可能</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">迷路循环</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
>     *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">图搜索</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：聪明，用内存换效率，</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">永不重复</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
> 4.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">评价算法</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：用</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">完备性、最优性、时间、空间</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">这四个标准来判断一个搜索算法的优劣。</span></span>

## **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">三、无信息搜索（Uninformed Search）</span></span>**

### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">核心算法详解：</span></span>**

#### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">1. 广度优先搜索（Breadth-First Search, BFS）</span></span>**

![\<img alt="" data-attachment-key="764E9B2A" width="578" height="507" src="attachments/764E9B2A.png" ztype="zimage"> | 578](attachments/764E9B2A.png)

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">策略</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：扩展最浅的未扩展节点</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">实现</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：FIFO队列</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">性质</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>

    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">完备性：是（当分支因子b有限时）</span></span>
    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">最优性：仅当所有单步代价相等时为是</span></span>
    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">时间复杂度：O(b^d)</span></span>
    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">空间复杂度：O(b^d)</span></span>

> 这应该是最简单的东西，按道理应该学过队列的人都知道这玩意，所以不过多赘述了，怕你觉得我唠叨

#### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">2. 一致代价搜索（Uniform Cost Search, UCS）、</span></span>**

![\<img alt="" data-attachment-key="MCS3KHTU" width="552" height="513" src="attachments/MCS3KHTU.png" ztype="zimage"> | 552](attachments/MCS3KHTU.png)

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">策略</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：扩展路径代价g(n)最小的节点</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">实现</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：按g(n)排序的优先队列</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">性质</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>

    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">完备性：是（当单步代价≥ε>0）</span></span>
    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">最优性：是</span></span>
    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">时间复杂度：O(b^(1+⌈C*/ε⌉))</span></span>
    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">空间复杂度：O(b^(1+⌈C*/ε⌉))</span></span>

> ```
> 按代价分层探索的“谨慎旅行者”
> ```
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">完备性与最优性</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：UCS非常可靠。只要每一步的成本不为零（有一个最小成本ε），并且存在解，它就一定能找到，而且找到的还是</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">成本最低</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">的解。这是它相比BFS的巨大优势。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">复杂度的独特之处</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：它的复杂度不直接依赖于深度</span></span>$d$，而是依赖于**最优解的总成本**$C^*$ 和 **最小单步成本**$ε$。
>
>     *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">可以把搜索空间想象成按“总花费”划分的层级：</span></span>
>
>         *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">第一层：所有花费 ≤ 1 * ε 的路径</span></span>
>
>         *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">第二层：所有花费 ≤ 2 * ε 的路径</span></span>
>
>         *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">...</span></span>
>
>         *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">第</span></span> $⌊C/ε⌋$ *<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">层：所有花费 ≤ C</span></span>*<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"> 的路径（包含最优解）</span></span>
>
>     *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">算法需要一层一层地探索，直到找到目标。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">核心思想</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：UCS探索的是</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">代价的边界</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，而不是深度的边界。它保证在找到更优解之前，绝不会去探索代价更高的路径。</span></span>

#### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">3. 深度优先搜索（Depth-First Search, DFS）</span></span>**

![\<img alt="" data-attachment-key="XSVZ25GN" width="571" height="504" src="attachments/XSVZ25GN.png" ztype="zimage"> | 571](attachments/XSVZ25GN.png)

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">策略</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：扩展最深的未扩展节点</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">实现</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：LIFO栈</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">性质</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>

    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">完备性：否（可能在无限深度空间失败）</span></span>
    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">最优性：否</span></span>
    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">时间复杂度：O(b^m)</span></span>
    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">空间复杂度：O(bm)</span></span>

> 同BFS，学过栈的人都学过这玩意，不讲

#### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">4. 迭代加深搜索（Iterative Deepening Search, IDS）</span></span>**

![\<img alt="" data-attachment-key="V6NTVI4Q" width="559" height="520" src="attachments/V6NTVI4Q.png" ztype="zimage"> | 559](attachments/V6NTVI4Q.png)

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">策略</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：重复进行深度限制递增的深度优先搜索</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">实现</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：外层循环增加深度限制，内层进行深度限制DFS</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">性质</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>

    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">完备性：是</span></span>
    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">最优性：当单步代价相等时为是</span></span>
    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">时间复杂度：O(b^d)</span></span>
    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">空间复杂度：O(bd)</span></span>

> ```
> “鱼与熊掌兼得”的聪明人
> ```
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">IDS做了一个惊人的权衡</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：它通过</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">重复计算</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">来换取</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">极低的内存消耗</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">它为什么完备且最优？</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"> 因为它本质上是在</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">模仿BFS</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。每次增加深度限制，它就像进行一次新的BFS，只不过是用DFS的方式实现的。因此它具备BFS的优点。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">时间复杂度的奥秘</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>
>
>     *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">看起来它不断重复计算，很浪费。但事实上，在搜索树中，</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">绝大多数的节点都在最底层</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
>     *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">计算量主要来自于最后一次迭代（深度=d的那次）。之前的迭代虽然重复，但它们的总计算量和最后一次相比是微不足道的。因此，它的时间复杂度和BFS同属一个级别，都是</span></span> $O(b^d)$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">空间复杂度：与DFS相同</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：因为它在每一轮迭代中使用的都是DFS，所以它继承了DFS最大的优点——</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">超低的内存占用</span></span>** $O(bd)$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">分析题</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：比较不同算法在特定问题上的适用性</span></span>

> 如何选择算法？
> ```
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">当你需要最优解，且单步代价不同时</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>
>
>     *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">选择UCS</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。它是BFS在非均匀代价情况下的通用升级版。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">当你内存非常有限，但又需要完备性和最优性时</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>
>
>     *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">选择IDS</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。它用稍微多一点的计算时间，换来了巨大的空间节省。是解决大规模问题的利器。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">当你只是需要快速找到一个解，而不关心是否最优，并且怀疑解在较深层次时</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>
>
>     *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">可以冒险选择DFS</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。它可能在短时间内给你一个惊喜，但也可能让你陷入漫长的等待。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">当单步代价均为1，且树不深，内存充足时</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>
>
>     *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">选择BFS</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。它简单直接。</span></span>
>

> ```
>
> 1.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">UCS (一致代价搜索)</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">可靠的老黄牛</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。保证找到最优解，不受单步代价不均的影响，但时空复杂度与最优解的成本挂钩。</span></span>
>
> 2.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">DFS (深度优先搜索)</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">赌徒/冒险家</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。内存开销极小，能找到深层的解，但不保证找到最优解，甚至可能完全找不到解。</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">用空间换时间（风险）</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
> 3.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">IDS (迭代加深搜索)</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">聪明的模仿者</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。通过重复计算来模仿BFS的行为，从而同时获得了</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">BFS的完备性/最优性</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">和</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">DFS的低空间开销</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。是内存紧张时的首选最优算法。</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">用时间换空间</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
> <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">在人工智能搜索中（当然你要打NOI比赛也是），你几乎总是在</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">时间</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">和</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">空间</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">之间进行权衡。理解这些算法的性能，就是为了在面临具体问题时，做出最明智的权衡。</span></span>

## **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">四、有信息搜索（Informed Search）</span></span>**

### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">核心概念详解：</span></span>**

#### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">1. 启发函数（Heuristic Function）</span></span>**

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">定义</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：h(n)=从节点n到目标节点的最小代价估计</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">要求</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：非负，目标节点h(n)=0，易于计算</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">例子</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>

    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">八数码：错位瓷砖数、曼哈顿距离</span></span>
    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">路径规划：直线距离</span></span>

> ```
> “盲人摸象” vs “有地图导航”
> ```
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">无信息搜索 (盲人摸象)</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：像BFS、UCS这样的算法，它们对目标在哪里</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">一无所知</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。它们只能机械地、均匀地向四周探索，或者根据</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">已经走过的路程(g(n))</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"> 来决定方向。效率低下。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">有信息搜索 (有地图导航)</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：像A*这样的算法，它有一张“地图”——</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">启发式函数h(n)</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。这个函数能告诉它“目标大概在哪个方向，大概有多远”。这使得它能</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">直奔目标</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，大大减少搜索范围，效率极高。</span></span>
>
> <!---->
>
> ```
> “缺德地图”——启发式函数
> ```
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">h(n)就是你的“导航”</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。比如你在一个乡村里找小卖部，你的导航地图（启发式）可能并非完全准确，它更类似于“朝着那边的村子里走”，或者用直线距离来估计大概还要走多远。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">关键要求</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>
>
>     1.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">永不高估</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：你的导航可以低估（保守估计），但绝不能高估。如果它觉得地铁站只有100米远，结果走了1公里，你就会走弯路。这就是“</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">可采纳性</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">”。</span></span>
>
>     2.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">快速计算</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：这个导航的估计距离必须是瞬间产生的。如果为了估计还要走多远，需要先规划一遍完整路线，那就本末倒置了。</span></span>

#### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">2. 贪心搜索（Greedy Search）</span></span>**

![\<img alt="" data-attachment-key="Q2FP9FF8" width="607" height="552" src="attachments/Q2FP9FF8.png" ztype="zimage"> | 607](attachments/Q2FP9FF8.png)

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">策略</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：扩展h(n)最小的节点</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">问题</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：不保证最优性，可能陷入局部最优</span></span>

> ```
> “只看终点，不顾起点”的冒险家
> ```
>
> *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">贪心搜索</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">只相信直觉 </span></span>** $h(n)$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，完全不顾及自己已经走了多远 </span></span> $g(n)$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">问题</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：它可能会被直觉误导，走向一个“看起来很近”但实际上需要绕远路的目标，或者陷入死胡同。如图中，它被节点 </span></span> $C$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">的低 </span></span> $h$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">值吸引，却走上了一条代价极高的路径。</span></span>

#### **<span style="color: rgb(255, 32, 32)"><span style="background-color: rgb(255, 255, 255)">3. A*搜索</span></span>**

![\<img alt="" data-attachment-key="HKGYL2LM" width="637" height="563" src="attachments/HKGYL2LM.png" ztype="zimage"> | 637](attachments/HKGYL2LM.png)

**<span style="background-color: rgba(255, 102, 102, 0.5)">（这幅图是A搜索！！！！不是A*搜索！！）</span>**

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">策略</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：扩展</span></span> $f(n)=g(n)+h(n)$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">最小的节点</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">可采纳性（Admissibility）</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span> $0≤h(n)≤h*(n)$

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">一致性（Consistency）</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span> $h(n)≤c(n,a,n′)+h(n′)$

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">定理</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：如果</span></span> $h(n)$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">可采纳，则A*最优</span></span>

### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">可采纳启发式构造方法：</span></span>**

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">放松问题法</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：通过放松原问题约束得到可采纳启发式</span></span>

    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">八数码放松1：允许瓷砖飞向目标位置→错位数启发式</span></span>
    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">八数码放松2：允许瓷砖重叠移动→曼哈顿距离启发式</span></span>

### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">启发式质量比较：</span></span>**

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">支配关系</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span> $h₂$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">支配</span></span> $h₁$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">如果</span></span> $∀n, h₁(n)≤h₂(n)≤h*(n)$

*   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">被支配的启发式通常能扩展更少节点</span></span>

### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">考试考查形式：</span></span>**

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">设计题</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：为给定问题设计可采纳启发式</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">判断题</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：判断给定启发式是否可采纳</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">证明题</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：证明A*在可采纳启发式下的最优性</span></span>

    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">关键步骤：证明在目标节点被选择前，最优路径上的节点会被扩展</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">计算题</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：给定搜索图，写出A*扩展顺序</span></span>

> ```
> “虽然兼顾，但直觉不准”————从贪心算法到A搜索
> ```
>
> *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">A搜索聪明地结合了 </span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">“已付出成本g(n)”</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"> 和 </span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">“预估成本h(n)”</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">问题</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：如果</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">启发式函数 </span></span>** $h(n)$  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">高估了</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">真实代价，它仍然可能错过最优路径。如图中，因为通往 </span></span> $B$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">的路径上有一个 </span></span> $h$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">值极高的节点，算法误以为那条路很糟，从而放弃了它，尽管它才是真正的最优路径。</span></span>
>
> <!---->
>
> ```
> 确保最优性的黄金法则————从A搜索到A*搜索
> ```
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">A*算法 = A搜索 + 可采纳启发式</span></span>**
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">可采纳启发式</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：就是上面提到的</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">永不高估</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">的直觉。数学表达为：对于所有节点n，</span></span> $0 ≤ h(n) ≤ h^*(n)$ ，其中 $h^*(n)$ 是真实的最近代价。
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">定理</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">只要使用可采纳启发式，A*算法保证能找到最优解！</span></span>**
>
> <!---->
>
> ```
> 如何设计可采纳的启发式？
> ```
>
> <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">这是一个设计启发式的万能方法。通过</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">放松原问题的规则</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，得到一个更简单的问题，这个简单问题的最优解成本，一定是原问题真实成本的</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">低估</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">八数码例子1</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：放松规则为“瓷砖可以飞到自己目标位置”，那么启发式就是 </span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">错位瓷砖的数量</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">（因为每步只能移动一个，实际步数肯定≥错位数）。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">八数码例子2</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：放松规则为“瓷砖可以穿过其他瓷砖”，那么启发式就是 </span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">曼哈顿距离之和</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">（因为实际移动中需要绕开其他瓷砖，实际步数肯定≥曼哈顿和）。</span></span>
>
> <!---->
>
> ```
> A*搜索的细节与优化
> ```
>
> #### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">何时终止？</span></span>**
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">错误方法（入队时停）</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：一旦把某个目标节点加入优先队列就宣布胜利。这可能导致找到</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">非最优解</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，因为队列里可能还有一条通往更优目标的路径还没被检查。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">正确方法（出队时停）</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：必须等到从优先队列中</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">取出</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">一个目标节点时，才能终止。因为这保证了此时这个目标的f值是最小的，不存在更优的解了。</span></span>
>
> #### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">最优性证明——为什么A*不会错过最优解？</span></span>**
>
> **证明的核心逻辑是：A\*在扩展一个次优目标G2之前，一定会先扩展最优路径上的某个节点n。**
>
> *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">因为</span></span> $G2$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">是次优的，所以它的总代价</span></span> $f(G2)$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">比最优解</span></span> $G1$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">的总代价</span></span> $g(G1)$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">要高。</span></span>
>
> *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">由于</span></span> $h(n)$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">是可采纳的（不高估），所以最优路径上节点n的</span></span> $f(n)$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">值不会超过</span></span> $g(G1)$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
> *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">因此，</span></span> $f(n) < f(G2)$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。在按f值排序的优先队列中，n会比G2</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">先被扩展</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
> *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">所以，A*在碰到次优解G2之前，早就沿着最优路径找到G1了。</span></span>
>
> #### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">弱点与改进</span></span>**
>
> **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">没有弱点吗？问题：A*消耗大量内存。</span></span>**
>
> *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">解决方案：迭代加深A*，束搜索。</span></span>

## **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">五、A*的改进变种</span></span>**

### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">核心算法详解：</span></span>**

#### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">1. 迭代加深A</span></span>*<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">（IDA</span></span>*<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">）</span></span>**

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">策略</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：重复进行f值限制递增的深度优先搜索</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">优势</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：保持A*最优性同时将空间复杂度降至O(bd)</span></span>

> ```
> 迭代加深A*（IDA）
> ```
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">核心思想</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：通过重复进行DFS来获得DFS的空间优势。与传统迭代加深搜索受限于第t次迭代的最大深度不同，这里只扩展满足 </span></span> $f(n) ≤ f_{t-1}^*$  的节点。 $f_{t-1}^*$  是上一次迭代中的最小代价。
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">实现</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>
>
>     *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">对于迭代次数 t = 0, 1, ...</span></span>
>
>     *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">执行一次深度优先搜索，但</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">仅扩展那些f值不超过当前阈值 </span></span>** $f_{t-1}^*$  的节点。
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">完备性</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：是</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">最优性</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：是</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">时间复杂度</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span> $O(b^m)$
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">空间复杂度</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span> $O(bd)$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">线性复杂度！</span></span>**
>
> <!---->
>
> ```
> “预算”不断增加的深度优先搜索
> ```
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">它是什么？</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"> IDA* 可以看作是一系列受控的深度优先搜索。</span></span>
>
>     *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">“预算”概念</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：想象你有一个搜索的“预算”（即f值阈值）。在第一轮，你设定一个很小的预算（比如起点的f值），然后进行一次DFS，但</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">绝不超支</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">——任何f值超过预算的节点，你直接忽略，并记下它们中超支最少的那个f值是多少。</span></span>
>
>     *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">增加预算</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：在下一轮，你把预算增加到上一轮记下的那个最小的超支f值，然后重新开始一次DFS。这次你可以探索得更深一些。</span></span>
>
>     *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">重复过程</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：如此反复，不断增加预算并重新搜索，直到你在一轮DFS中成功找到了目标节点。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">为什么它如此出色？</span></span>**
>
>     1.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">空间效率极高</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：因为它本质上是DFS，同一时间只存储一条路径上的节点，所以空间复杂度是</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">O(深度)</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，这对于搜索大树来说是巨大的优势。</span></span>
>
>     2.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">保持最优性</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：因为它最终会模拟A</span></span>*<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">的行为，并且f值阈值最终会增加到最优解的f值，所以它和A</span></span>*<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">一样，能保证找到最优解。</span></span>
>
>     3.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">代价</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：主要缺点是</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">时间</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。因为它每一轮都几乎是从头开始，所以会有大量的</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">重复计算</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。但它用“时间”换来了极其宝贵的“空间”。</span></span>

#### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">2. 束搜索（Beam Search）</span></span>**

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">策略</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：只保留优先级队列中前k个节点</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">优势</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：空间复杂度O(k)，但牺牲完备性和最优性</span></span>

> ```
> 束搜索
> ```
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">核心思想</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：将优先队列的大小固定为k。只保留前k个最优的节点。束搜索是一种通用技术，并非只适用于A*。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">实现</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：边界是一个大小为k的优先队列，按 </span></span> $g(n) + h(n)$  排序，**丢弃掉最大的那些节点**。
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">完备性</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：否</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">最优性</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：否</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">时间复杂度</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span> $O(b^m)$
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">空间复杂度</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span> $O(k)$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">常数复杂度！我的天哪束搜索大人，吓哭了</span></span>**
>
> <!---->
>
> ```
> “优中选优”的贪婪策略
> ```
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">它是什么？</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"> 束搜索是一种非常“吝啬”的搜索算法。它不像A*那样保留所有待探索的节点，而是</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">只保留每一层它认为“最好”的k个节点</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">（k是你事先设定的一个数，比如10、100），然后把其他节点全部无情地丢弃。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">工作原理</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：想象你在进行BFS，但每一层扩展后，你不是保留所有新生成的子节点，而是</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">只保留其中优先级最高的k个</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，然后基于这k个节点继续扩展下一层。</span></span>
>
> **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">巨大的优势与致命的代价：</span></span>**
>
> 1.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">无与伦比的空间效率</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：因为队列大小被固定为k，所以它的空间复杂度是</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">O(k)</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，这是一个</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">常数</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">！这意味着无论问题多复杂，它的内存占用都是稳定且可控的。</span></span>
>
> 2.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">致命的缺点</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>
>
>     *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">不完备</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：由于它可能过早地丢弃了包含解的分支，所以</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">即使解存在，它也可能找不到</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
>     *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">不最优</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：它找到的解很可能是次优的，因为通往最优解路径上的节点可能在某一层因为不在前k名而被丢弃了。</span></span>
>
> **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">应用场景</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：正因为有这些缺点，束搜索不用于要求绝对最优和可靠性的场景（如机器人导航、医疗诊断）。但它广泛应用于那些</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">“有一个较好解比没有解要好”</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，或者</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">“需要极快给出一个还不错的结果”</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"> 的领域，例如：</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">机器翻译</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：在生成一句话时，每一步都只保留几个最可能的单词选择。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">语音识别</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：在识别一段语音时，只保留几条最可能的识别路径。</span></span>
>

> 关键知识点回顾与解题思路构建
> ```
>
> #### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">回顾的核心知识点：</span></span>**
>
> **1. <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">A*搜索算法基础</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>$f(n) = g(n) + h(n)$
>
> > **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">f(n) = g(n) + h(n)</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"> 是A*搜索算法的核心评估函数，其中：</span></span>
> >
> > ```
> > g(n) - 从起点到当前节点的实际代价
> > ```
> >
> > *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">定义</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：从初始状态到达节点n已经花费的实际成本</span></span>
> >
> > *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">计算方式</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：路径上所有步骤代价的累加和</span></span>
> >
> > *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">特性</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>
> >
> >     *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">是确定性的、已知的数值</span></span>
> >     *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">随着搜索的进行不断更新</span></span>
> >     *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">保证找到的路径是实际可行的</span></span>
> >
> > **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">示例</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：在网格路径规划中，g(n)就是从起点到当前节点走过的网格数总和。</span></span>
> >
> > ```
> > h(n) - 从当前节点到目标的估计代价（启发函数）
> > ```
> >
> > *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">定义</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：对从节点n到目标节点最小代价的估计</span></span>
> >
> > *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">计算方式</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：使用启发式函数估算</span></span>
> >
> > *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">关键要求</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>
> >
> >     *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">可采纳性(Admissibility)</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：h(n) ≤ h*(n)，即永不高估真实代价</span></span>
> >
> >     *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">一致性(Consistency)</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：h(n) ≤ c(n,n') + h(n')，满足三角不等式</span></span>
> >
> > **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">示例</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：曼哈顿距离、欧几里得距离等。</span></span>
> >
> > ```
> > f(n) - 通过节点n的路径总代价估计
> > ```
> >
> > *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">意义</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：平衡了"已经付出的代价"和"预计还要付出的代价"</span></span>
> >
> > *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">作用</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：决定节点在优先队列中的扩展顺序</span></span>
> >
> > *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">最优性保证</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：当h(n)可采纳时，A*保证找到最优路径</span></span>
>
> **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">2. 迭代加深思想</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：从深度限制开始，逐步增加深度</span></span>
>
> > **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">迭代加深(Iterative Deepening)</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"> 是一种结合了DFS空间效率和BFS完备性的搜索策略。</span></span>
> >
> > ```
> > 核心思想：
> > ```
> >
> > *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">不是一次性搜索到最大深度</span></span>
> >
> > *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">而是</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">逐步增加深度限制</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，在每个限制下进行完整的深度优先搜索</span></span>
> >
> > *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">每次迭代都重新开始搜索，但深度限制增加</span></span>
> >
> > ### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">工作流程：</span></span>**
> >
> > ```
> > 深度限制 = 0
> > while 未找到解:
> >     进行深度限制为"深度限制"的深度优先搜索
> >     if 找到解:
> >         返回解
> >     else:
> >         深度限制 += 1
> > ```
> >
> > ### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">具体执行过程：</span></span>**
> >
> > #### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">第一次迭代：深度限制=0</span></span>**
> >
> > ```
> > 只检查起点本身
> > A
> > ```
> >
> > #### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">第二次迭代：深度限制=1</span></span>**
> >
> > ```
> > 检查深度1的所有节点
> > A
> > ├── B
> > ├── C
> > └── D
> > ```
> >
> > #### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">第三次迭代：深度限制=2</span></span>**
> >
> > ```
> > 检查深度2的所有节点
> > A
> > ├── B
> > │   ├── E
> > │   └── F
> > ├── C
> > │   └── G
> > └── D
> >     ├── H
> >     └── I
> > ```
>
> **3. <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">IDA*的特殊性</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：使用f值作为深度限制，而非简单深度</span></span>
>
> **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">4. 空间复杂度优势</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：相比标准A</span></span>*<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">的</span></span>*$O(b^d)$*<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，IDA</span></span>*<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">为</span></span>$O(d)$
>
> **解题思路构建：**
>
> <span style="color: rgb(15, 17, 21)">标准A*算法问题 → 内存消耗过大 → 引入迭代加深思想 →</span>
>
> <span style="color: rgb(15, 17, 21)">使用f值作为阈值 → 深度优先搜索 + 阈值递增 →</span>
>
> <span style="color: rgb(15, 17, 21)">实现IDA*算法</span>
>

## **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">一、对抗性搜索概述（Adversarial Search Overview）</span></span>**

#### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">2. 博弈的形式化定义</span></span>**

```
States: S (起始状态 s₀)
Players: P = {1...N}
Actions: A (依赖于玩家和状态)
Transition: S × A → S
Terminal Test: S → {true, false}
Utilities: S × P → R (效用函数)
```

## **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">二、Minimax算法详解</span></span>**

### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">核心概念详解：</span></span>**

#### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">1. 基本思想</span></span>**

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">MAX玩家</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：选择使效用最大化的动作</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">MIN玩家</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：选择使效用最小化的动作</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">递归计算</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：从叶子节点向上传播效用值</span></span>

#### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">2. 算法性质</span></span>**

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">完备性</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：是（如果树有限）</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">最优性</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：是（对抗理性对手）</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">时间复杂度</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span> $O(b^m)$

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">空间复杂度</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span> $O(bm)$

> <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">可以把 Minimax 算法理解为：<br></span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">在一个“你走一步，我走一步”的游戏中，你（MAX）假设对手（MIN）是世界上最聪明的人。为了不输，你必须预判他的所有预判。你递归地思考所有可能的未来，最终选择一个能保证在最坏情况下也能获得最好结果的走法。</span></span>**
>
> <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">它的核心是</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">理性决策</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">和</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">递归计算</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，但它的缺点是</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">计算量太大</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，因此在复杂游戏中必须结合其他技术才能实用。下面我结合PPT中的内容帮助大家简单理解一下这一部分：</span></span>
>
> ```
> 核心思想 - 递归博弈
> ```
>
> <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">想象你在和下棋，你走一步，对手走一步，交替进行。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">你（MAX玩家）</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：你想让分数（效用）</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">越高越好</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。所以你在你的回合，会选择所有可能动作中，能带来</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">最大</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">最终分数的那个动作。这就是 </span></span>`max-value` 函数。
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">对手（MIN玩家）</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：他想让分数</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">越低越好</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">（因为这是零和游戏，你得分高就是他得分低）。所以在他的回合，他会选择所有可能动作中，能带来</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">最小</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">最终分数的那个动作。这就是 </span></span>`min-value` 函数。
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">关键点</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：这两个函数是</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">互相递归调用</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">的。当你(</span></span>`max`)在计算一个动作的分数时，你必须考虑对手(`min`)接下来会怎么走；而对手(`min`)在计算时，又会假设你(`max`)之后会走最优的一步。如此递归下去，直到棋局结束。
>
> <!---->
>
> ```
> 代码优化与清晰化
> ```
>
> <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">这个版本是上一个版本的优化，更清晰，更容易实现。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">核心是一个“分发器”函数 </span></span>**`value(state)`：它像一个交通警察，根据“当前轮到谁了”来决定应该调用 `max-value` 还是 `min-value`，或者如果游戏已经结束，就直接返回胜负平局的分数（效用值）。
> *   `max-value` 和 `min-value` 函数不需要知道下一个该调用谁，它们只负责做好自己的事情（取最大值或最小值），然后统一调用 `value` 函数，让分发器去决定下一步的逻辑。这减少了代码的重复和耦合。
>
> <!---->
>
> ```
> 看图说话
> ```
>

>
> <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">我们假设这个数字序列是树底层（叶子节点）的游戏结果分数。</span></span>
>
> 1.  <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">从最底层开始（比如分数 3, 12, 8, 2, 4, 6, 14, 5, 2）。</span></span>
>
> 2.  <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">往上一层，假设是 MIN 玩家的回合。MIN 会在他的子节点中选</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">最小值</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
>     *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">比如，他面对 {3, 12, 8}，会选 </span></span>`min(3,12,8) = 3`。

>
>     *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">他面对 {14, 5, 2}，会选 </span></span>`min(14,5,2) = 2`。
>
> 3.  <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">再往上一层，轮到 MAX 玩家。MAX 会在他面对的选择中选</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">最大值</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
>     *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">他面对 {3, 2, 2}（来自上一层MIN的选择结果），会选 </span></span>`max(3,2,2) = 3`。
>
> 4.  <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">所以，根据 Minimax，根节点的值（对于MAX玩家而言）是 </span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">3</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，他应该选择通向这个值为3的路径的第一步。</span></span>
>

> ```
> 算法的优缺点
> ```
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">优点</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：Minimax 提供了一个</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">理论上的最优策略</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。只要你和一个“永不犯错”的理性对手玩，遵循Minimax策略能保证你获得至少可能获得的最好结果（平局或胜利）。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">缺点</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：如图中所示，如果对手</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">不完美（会犯错）</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，Minimax 可能会错失良机。</span></span>
>
>     *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">例子</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：MAX 在顶层做选择。右边的分支通向一个值为9的节点，虽然它现在看起来比左边的10要差，但它下面隐藏着一个巨大的机会（100分）。但如果对手(MIN)是理性的，他绝不会让你得到那100分，他会在他的回合选择9，从而封死这条路。所以Minimax会保守地选择左边稳妥的10。</span></span>
>
>     *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">但是</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，如果对手是个菜鸟，在那个节点上没有选择9，而是错误地选择了通向100的路径，那你就能大获全胜。Minimax 因为假设对手完美，所以</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">不会去冒这个险</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。在现实中，对抗会犯错的对手，有时需要更“冒险”的策略。</span></span>
>
> <!---->
>
> ```
> 为什么需要改进
> ```
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">复杂度爆炸</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：Minimax 需要遍历整个博弈树。</span></span>
>
>     *   $b$  是**分支因子**（平均每步有多少种走法）。
>
>     *   $m$  是**最大深度**（一盘游戏大概走多少步）。
>
>     *   $O(b^m)$  是一个**指数级**的增长。比如国际象棋，35^100 是一个天文数字，即使全世界的计算机算到宇宙毁灭也算不完。
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">空间复杂度</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"> </span></span> $O(bm)$  相对较好，因为它是以深度优先的方式搜索的，同一时间只需要存储一条路径上的节点。
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">结论</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：正是由于这种恐怖的效率问题，我们才需要后续的优化技术，比如：</span></span>
>
>     1.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">Alpha-Beta 剪枝</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：在不影响结果的前提下，“剪掉”明显不好的分支，避免计算它们。</span></span>
>
>     2.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">深度限制</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：不搜索到最终结局，而是在一定深度后，用一个</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">评估函数</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">来估算当前局面的好坏（比如，用“我方皇后数量 - 对方皇后数量”来粗略估计胜负概率）。</span></span>
>
>     3.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">蒙特卡洛树搜索（MCTS）</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：通过随机模拟来智能地选择有希望的分支进行深入探索，而不是均匀地探索所有分支。</span></span>

## **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">三、Alpha-Beta剪枝详解</span></span>**

### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">核心概念详解：</span></span>**

#### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">1. 剪枝原理</span></span>**

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">α值</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：MAX玩家在当前路径能保证的最佳值</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">β值</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：MIN玩家在当前路径能保证的最佳值</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">剪枝条件</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>

    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">在MIN节点：如果v ≤ α，则剪枝</span></span>
    *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">在MAX节点：如果v ≥ β，则剪枝</span></span>

#### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">3. 剪枝效果</span></span>**

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">最优性保证</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：不影响根节点的决策</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">效率提升</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：最佳情况下时间复杂度降至O(b^(m/2))</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">节点顺序</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：好的动作顺序能显著提高剪枝效率</span></span>

### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">考试考查形式：</span></span>**

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">图示题</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：在博弈树上标出剪枝发生的位置</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">计算题</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：跟踪α、β值的更新过程</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">简答题</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：解释为什么α-β剪枝不影响最优决策</span></span>

> **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">Alpha-Beta 剪枝是 Minimax 算法的“智能跳过”机制。</span></span>**
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">核心</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：它通过跟踪两个边界值 </span></span>`α`和 `β`，在搜索过程中一旦发现某条分支“显然”不会被任何一方选择，就立即停止对该分支的深入计算。
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">好处</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">极大地减少了计算量</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，但</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">不改变最终的决策结果</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">关键</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：理解“谁关心这个节点的值”以及“什么时候这个值已经坏到不会被选择”。</span></span>
>

>
> ```

> 警察与小偷的比喻
> ```
>
> <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">把下棋想象成一个</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">警察(MAX)</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">抓</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">小偷(MIN)</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">的游戏。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">α (Alpha)</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：是</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">警察的“底线”</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。代表警察目前已经找到的一条路，在这条路上他</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">至少</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">能保证抓到小偷时，小偷的狼狈程度是 </span></span> $α$ （分数越高警察越开心）。警察想提高这个底线。
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">β (Beta)</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：是</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">小偷的“底线”</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。代表小偷目前已经找到的一条路，在这条路上他</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">至少</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">能保证自己的狼狈程度不会超过 </span></span> $β$ （分数越低小偷越开心）。小偷想降低这个底线。
>
> **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">剪枝什么时候发生？</span></span>**
>
> 1.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">在小偷(MIN)的地盘</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：当小偷发现某条路会让他变得比警察的底线 </span></span> $α$  还要狼狈（ $v ≤ α$ ），他会说：“这路太坑了，比警察已知的最好情况还糟，我肯定不会走这！剩下的岔路不用看了。” → **β 剪枝**。
>
> 2.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">在警察(MAX)的地盘</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：当警察发现某条路会让他得到的成绩比小偷的底线 </span></span> $β$  还要差（ $v ≥ β$ ），他会说：“这路不行啊，比小偷能接受的最坏情况还好，他肯定会把我引到别的路上去，这路白费劲了，剩下的岔路不用看了。” → **α 剪枝**。
>

> <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">代码和 Minimax 非常像，只多了三样东西：</span></span>

>
> 2.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">剪枝检查</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>

>     *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">在 </span></span>`max-value` 中，如果发现 `v >= β`，就立即返回，进行 **β 剪枝**。
>
>     *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">在 </span></span>`min-value` 中，如果发现 `v <= α`，就立即返回，进行 **α 剪枝**。
>

>
>     *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">在 </span></span>`max-value` 中，用找到的更大值更新 `α`（提高警察的底线）。
>
>     *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">在 </span></span>`min-value` 中，用找到的更小值更新 `β`（降低小偷的底线）。
>

> 重要启示
> ```
>
> 1.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">正确性保证</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：最核心的一点！剪枝只是跳过了计算，但最终给根节点（你当前要做的决策）算出的值，和完整计算 Minimax 得到的结果是</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">一模一样</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">的。你不会因为剪枝而选错。</span></span>
>
> 2.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">副作用</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：因为你跳过了计算，所以那些被剪枝分支的父节点，你只知道它的值“不会好于/坏于某个数”，但不知道它的精确值。这通常没问题，因为我们只关心根节点的决策。</span></span>
>
> 3.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">排序很重要</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：如果我们能聪明地</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">先搜索看起来最有希望的子节点</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">（比如在MAX节点先搜估值高的），就能更早地更新 </span></span>`α` 或 `β`，从而更早地触发剪枝，跳过更多不必要的计算。这能极大提升效率。
>
> 4.  **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">威力巨大</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：效率从 </span></span> $O(b^m)$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"> 提升到 </span></span> $O(b^{m/2})$ <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，意味着在相同时间内，你能思考的步数（搜索深度）大约是原来的两倍。这在游戏中是巨大的优势。</span></span>

## **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">四、蒙特卡洛树搜索（MCTS）详解</span></span>**

### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">核心概念详解：</span></span>**

#### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">1. MCTS四阶段</span></span>**

#### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">2. UCB选择公式</span></span>**

$\text{UCB} = v_i + C × \sqrt{\ln N / n_i}$

<span style="color: rgb(15, 17, 21)">其中： </span>$v_i$<span style="color: rgb(15, 17, 21)"> = 节点i的胜率估计 </span>$n_i$<span style="color: rgb(15, 17, 21)"> = 节点i的访问次数 </span>$N =$ <span style="color: rgb(15, 17, 21)">父节点总访问次数 </span>$C =$<span style="color: rgb(15, 17, 21)"> 探索参数（通常为√2）</span>

#### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">3. 算法优势</span></span>**

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">无需领域知识</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：只需要游戏规则</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">非对称生长</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：聚焦有希望的分支</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">任意时间算法</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：随时可返回当前最佳动作</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">处理大规模状态</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：适合如围棋等复杂游戏</span></span>

### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">考试考查形式：</span></span>**

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">简答题</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：描述MCTS的四个阶段及其作用</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">计算题</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：给定访问次数，计算各节点的UCB值</span></span>

*   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">比较题</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：比较MCTS与minimax的优缺点</span></span>

> **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">MCTS/UCT 是一个“在战争中学习战争”的智能算法。</span></span>**
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">它不依赖任何人类知识</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">（比如象棋开局库、围棋定式），只需要知道游戏规则。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">它的核心是一个四步循环</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">选择</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">（用UCB公式智能选路）→ </span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">扩展</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">（开辟新节点）→ </span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">模拟</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">（随机快速对局）→ </span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">回溯</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">（更新节点统计数据）。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">通过成千上万次的循环</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，它像一个人一样，从零开始，通过反复试验和自我对弈，逐渐学会哪些走法是好的，哪些是坏的。最终，它会选择根节点下</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">访问次数最多</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">（而非胜率最高，因为访问次数综合反映了胜率和可靠性）的那个子节点作为实际落子动作。</span></span>
>
> <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">正是这套方法，让AI在没有人类先验知识的情况下，在围棋这种复杂度极高的游戏中，超越了千年积累的人类智慧。</span></span>
>
> ```
>  核心思想：用随机抽样估算整体
> ```
>
> <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">想象一下，你要计算一个画在操场上的不规则图形的面积。你没有公式，怎么办？<br>一个“蒙特卡洛”式的笨办法是：蒙上眼睛，朝整个操场随机扔一大把豆子。然后，数一数有多少颗豆子落在了图形内。</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">（图形内豆子数 / 总豆子数）</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"> 就约等于 </span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">（图形面积 / 操场面积）</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。扔的豆子越多，估算就越准。<br></span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">它的精髓就是：通过大量的随机试验，用统计结果来逼近一个复杂问题的答案。</span></span>**
>
> ```
> 蒙特卡洛树搜索 (MCTS) 四步循环
> ```
>
> <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">MCTS把蒙特卡洛的“随机试验”思想用在了博弈树上。它不是像Minimax那样暴力计算所有分支，而是</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">用模拟对局的方式，智能地、有侧重地探索这棵树</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。这个过程分为四步，循环进行。</span></span>
>
> #### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">第一步：选择</span></span>**
>
> **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">原文:</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"><br>Selection<br>Starting at root node R, recursively select optimal child nodes (explained soon) until a leaf node L is reached.</span></span>
>
> **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">翻译与解释:</span></span>**
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">翻译</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">: </span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">选择</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"> - 从根节点R开始，递归地选择</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">最优</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">的子节点，直到抵达一个叶节点L。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">在已知和未知之间谨慎前行：</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">你不是随机选，而是用一个聪明的公式（后面会讲）来选择。这个公式能平衡“</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">去探索没怎么去过的区域</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">”（探索）和“</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">多去胜率高的地方看看</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">”（利用）。你从树根开始，根据这个公式一路向下，直到走到一个你还没有完全展开的“前沿”节点。</span></span>
>
> #### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">第二步：扩展</span></span>**
>
> **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">原文:</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"><br>Expansion<br>If L is a not a terminal node (i.e. it does not end the game) then create one or more child nodes and select one C.</span></span>
>
> **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">翻译与解释:</span></span>**
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">翻译</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">: </span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">扩展</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"> - 如果叶节点L不是终止节点（即游戏还未结束），那么就为其创建一个或多个子节点，并从中选择一个节点C。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">迈出未知的一步：</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">你现在到达的“前沿”节点，可能下面还有你没见过的局面。你就在这个节点下，新开辟一个（或几个）可能的下一步，作为新的叶节点。比如，你选择其中一个新节点C，作为本次模拟的起点。</span></span>
>
> #### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">第三步：模拟</span></span>**
>
> **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">原文:</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"><br>Simulation<br>Run a simulated roll out randomly from C until a result is achieved.</span></span>
>
> **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">翻译与解释:</span></span>**
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">翻译</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">: </span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">模拟</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"> - 从节点C开始，运行一次随机模拟（快速走子），直到得到一个游戏结果（输/赢/平）。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">“撒一把豆子”：</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">这是最“蒙特卡洛”的一步！从C点开始，你和对手都完全随机地、快速地乱走，直到这盘棋下完，分出胜负。</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">这就像是在验证“从C点开始乱下，最终赢的概率有多大”</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。这次模拟的结果（比如赢了得1分，输了得0分）就是本次试验的样本。</span></span>
>
> #### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">第四步：回溯</span></span>**
>
> **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">原文:</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"><br>Backpropagation<br>Update the current move sequence with the simulation result.<br>Each node must contain two important pieces of information: an estimated value based on simulation results and the number of times it has been visited.</span></span>
>
> **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">翻译与解释:</span></span>**
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">翻译</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">: </span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">回溯</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"> - 用模拟的结果更新当前路径上的节点序列。每个节点必须包含两个重要信息：基于模拟结果的估计值，以及它被访问的次数。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">记下这次的经验</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">在已知和未知之间谨慎前行</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">模拟结束后，你得到了一个分数（比如赢了：1分）。现在，你沿着刚才选择-扩展的路径</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">原路返回</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，从节点C一直回溯到根节点R。</span></span>
>
>     *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">对于路径上的</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">每一个节点</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，你都要做两件事：</span></span>
>
>         1.  <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">把它的</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">总收益</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">加上刚才的得分（比如赢了就+1）。</span></span>
>
>         2.  <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">把它的</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">访问次数</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">加1。</span></span>
>
>     *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">这样，每个节点就记录了两个关键数据：</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">总收益</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"> 和 </span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">访问次数</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。用前者除以后者，就得到了这个节点的</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">平均胜率</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
> <!---->
>
> ```
> UCB公式：聪明的决策指南
> ```
>
> $$
> \text{UCB} = v_i + C × \sqrt{\ln N / n_i}
> $$
>
> <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">这个公式就是在“选择”阶段用来决定走哪个子节点的。它计算一个分数，分数高的节点被选中。<br></span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">这个分数由两部分组成：</span></span>**
>
> 1.  $v_i$  (利用项)：这个节点的**平均胜率**。胜率越高，分数自然越高。这鼓励程序去走它认为好的棋。
>
> 2.  $C × \sqrt{\ln N / n_i}$ (探索项)：
>
>     *   $N$  是父节点的总访问次数。
>
>     *   $n_i$  是这个子节点被访问的次数。
>
>     *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">如果某个子节点</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">访问次数很少</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">（</span></span> $n_i$ 小），那么  $\sqrt{\ln N / n_i}$   的值就会很大，导致**探索项**变大。
>
>     *   <span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">这就会鼓励程序去尝试那些它还不怎么了解的节点，防止错过隐藏的好棋。</span></span>
>
> $C$ 是一个调节旋钮，用来控制你有多“好奇”。$C$ 越大，就越喜欢探索新节点。
>
> ```
> 从广撒网到精耕细作
> ```
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">初期</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：树还没怎么建，所有节点访问次数都很少。此时UCB公式中的“探索项”权重很大，算法会像好奇宝宝一样到处尝试，</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">广撒网</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。</span></span>
>
> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">后期</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：经过成千上万次模拟后，大部分节点的访问次数都上来了，“探索项”变小。此时，算法的决策主要依赖于“利用项”（胜率），它会</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">精耕细作</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">，把时间花在它已经确认的几条高胜率路线上。</span></span>
>
> <!---->
>
> ```
> 给算法起个正式的名字
> ```
>
> **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">UCT</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)"> 就是我们所讨论的这个完整算法的学术名称。它明确指出，这个算法就是 </span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">MCTS的框架 + UCB的选择策略</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">。你现在所理解的这个四步循环、用UCB公式选择节点的算法，就是UCT算法。</span></span>

> #### **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">关键知识点回顾：</span></span>**

> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">MCTS四步循环</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：选择（Selection）、扩展（Expansion）、模拟（Roll out）、回溯（Backpropagation）。</span></span>

> *   **<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">UCB公式</span></span>**<span style="color: rgb(15, 17, 21)"><span style="background-color: rgb(255, 255, 255)">：用于选择阶段，平衡探索和利用。公式为：</span></span>

>     \text{UCB}=\frac{w_i}{n_i}+C×\sqrt{\frac{\ln ⁡N}{n_i}}
>     $$
