各章节知识点
01

VR 定义与层级

  • 核心定义：实时仿真 + 多感官交互
  • 感知通道：视觉/听觉/触觉/嗅觉/味觉
  • 被动型（Passive）
  • 探索型（Exploratory）
  • 交互型（Interactive）
  • 协作型（Collaborative）
02

人机交互界面（HCI）

  • 输入设备：感知用户行为
  • 输出设备：向用户传递反馈
  • 感知循环：see → analyse → action
  • 各感知通道技术难点：
  • 视觉：渲染容易，理解难
  • 触觉：设备具有侵入性（Intrusive）
03

VR 硬件发展史（时间线）

  • 1838 立体镜（Stereoscopes）
  • 1962 Sensorama — 首款 VR 视频机
  • 1968 Sutherland — "Ultimate Display"
  • 1985 NASA VIVED / 1988 VIEW
  • 1980s末 EyePhones — 首款商用 HMD
  • 1992 CAVE；1995 Virtual Boy
  • 2012–2018 VR 复兴（Oculus 等）
04

VR 系统四大类型

  • 沉浸式 VR：HMD + 头部追踪，360° 视野
  • 非沉浸式 VR：普通显示器，可用立体眼镜
  • 增强现实 AR：透明眼镜叠加虚拟信息
  • 远程呈现：双向通信控制远端机器人
05

位置传感器（追踪方式对比）

  • 2D 鼠标：廉价，难用于 3D
  • 机械追踪器：精确，但范围受限
  • 超声波追踪器：廉价，精度低，受噪声干扰
  • 磁场追踪器：精度较高，受金属干扰
06

系统延迟（Latency）

  • 定义：头部移动 → 画面更新的总时间差
  • 来源：传感器 → 追踪计算 → 数据库更新 → 渲染
  • 影响：视觉与预期不一致 → 不适/晕动症

各章节知识点
01

追踪基础与坐标变换

  • DOF（自由度）：需追踪的独立运动维度
  • 变换公式：v2D = Mproj × Mview × Mworld × v3D
  • Mworld：模型坐标 → 世界坐标
  • Mview：世界坐标 → 视角坐标
  • Mproj：3D → 2D 投影
  • Mview 推导：R(roll/pitch/yaw) × T(eye)
02

四大性能指标（重要考点）

  • Accuracy：实际位置 vs 测量值的接近度
  • Jitter：静止时输出的随机波动
  • Drift：误差随时间累积增大
  • Latency：动作到信号接收的时间差
  • 大延迟 → Simulation Sickness
03

机械追踪器

  • 原理：关节运动链，电位计测量角度
  • 代表：Gypsy（42 个电位计，15 关节）
  • 优：精确、延迟小、无视线限制
  • 缺：侵入性、范围受限、沉重
04

磁场追踪器（AC vs DC）

  • 原理：固定发射器 + 移动接收器，9 个电压值
  • AC 问题：涡流（Eddy Currents）扭曲磁场
  • DC 改进：脉冲磁场 + 时延，让涡流消散
  • 误差公式：Error = K × d⁴
  • 优：无视线限制；缺：受金属干扰
05

超声波 / 光学 / 惯性追踪器

  • 超声波：飞行时间法；需直接视线；受温度影响
  • 光学-外部观察（Outside Looking In）：高精度，遮挡是难题
  • 光学-内部观察（Inside Looking Out）：HiBall，2000次/秒
  • 惯性：陀螺仪短期好+加速度计长期好互补
  • 陀螺误差 ∝ 时间；加速度计误差 ∝ 时间²
06

手势设备（传感原理对比）

  • Pinch Gloves：导电纤维，检测接触/断开
  • 5DT Data Gloves：光纤弯曲传感，光强变化
  • Didjigloves：电容弯曲传感，极板面积变化
  • CyberGloves：应变片差分电压，18/22 传感器

各章节知识点
01

人类视觉系统

  • 视网膜：1.26 亿光感受器，分布不均
  • 中央凹（Fovea）：高分辨率/彩色区域
  • 三种视锥：L(红)/M(绿)/S(蓝)；绿色最敏感
  • FOV：单眼 150°×135°；双眼 180°×135°；双目重叠 120°
  • 视网膜显示所需分辨率：9000×8100/眼
02

深度感知线索

  • 近距（0–30m）：会聚、调节、视差、运动视差
  • 远距（>30m）：遮挡、透视、雾霾、表面纹理
  • 会聚-调节冲突：HMD 中两者指向不同距离
  • → 视疲劳，VR 设计难题
03

HMD 光学成像

  • 镜头作用：放大 + 短距聚焦
  • 高斯薄透镜：1/d + 1/d' = 1/f
  • 放大倍数：M = f / (d' − f)
04

个人显示设备三类

  • HMD：图像投影于 1–5m 处，含光学放大
  • HHD：手持，按需观看（Virtual Binoculars）
  • 自动立体：无需眼镜；列交错；水平分辨率减半
  • 被动式：固定视区
  • 主动式：头追踪，动态视区
05

大容量显示

  • 快门眼镜系统：120–140 Hz；交替遮挡左/右眼
  • LCD 投影仪：3 独立 RGB 面板；清晰但有颗粒感
  • DLP 投影仪：DMD 微镜 ±10° 偏转；更平滑
  • CAVE：多面投影，完全沉浸
  • PanoWall：3 台拼接，25% 重叠，2D LUT 混合
  • V-Dome：360° 球幕，需预畸变图像
06

体积显示

  • 360° 全方位视图，无需眼镜
  • 原理：旋转屏幕 + 高速帧序列（5000 fps）
  • 分辨率：768×768×198 层
  • 典型案例：Perspecta（TI DMD + 旋转镜）

Module 4 · Output Devices — Sound & Haptics

VR 的"非视觉"感知模块，分为声音和触觉两大部分。声音部分：从声音物理性质 → 人耳定位机制 → HRTF 数学模型 → 实际 3D 声音系统；触觉部分：从皮肤感受器 → 触觉/力反馈分类 → 各类硬件设备。两条并行线索，都遵循"生理机制 → 技术实现"的逻辑。

各章节知识点
01

声音基础

  • 声音：压力波（Pressure Wave）
  • 声速公式：c = √(K/ρ)；空气中约 343 m/s
  • 波动特性：反射、折射、衍射
  • 人耳可听范围：20 Hz – 20 kHz
  • 最敏感频率：3–4 kHz（耳道放大 3–12 kHz）
02

3D 声音定位

  • 方位（Azimuth）：ITD（时间差）+ IID（强度差）
  • ITD：低频有效；低于 250Hz 时失效
  • IID（头影效应）：高频更明显
  • 仰角（Elevation）：耳廓反射引起频率变化
  • 距离（Distance）：先验知识 + 运动视差
03

HRTF 与 3D 声音系统

  • HRTF：声源到耳朵的频域传递函数
  • 公式：Yᵢ = Hᵢ ⊗ S（卷积运算）
  • 每人唯一；使用他人 HRTF → 误差率 30%
  • Convolvotron（1988）：首款 3D 声音系统
  • 扬声器方案：需矩阵求逆，须在甜区（Sweet Spot）
04

皮肤感受器

  • Meissner + Merkel：表面感受器
  • Pacinian + Ruffini：深层感受器
  • 慢适应（Merkel+Ruffini）：0–10 Hz，静态力
  • 快适应（Meissner+Pacinian）：50–300 Hz，变化
  • 指尖分辨率 2.5mm；手掌 11mm
05

触觉反馈设备

  • 触觉鼠标（iFeel）：振动触觉反馈
  • CyberTouch：6 个振动执行器，偏心配重电机
  • 温度手套：热电热泵，N/P 半导体，PID 控制
  • 本体感觉：髋关节 0.2°（最精）→ 脚趾 6.1°
06

力反馈设备

  • 力反馈摇杆（WingMan）：廉价，力矩反馈
  • PHANToM：3 主动 + 3 被动 DoF；医疗/科研
  • Haptic Master：3DoF，250N，刚度高但惯性大
  • CyberGrasp：手指独立力，最大 16N，含外骨骼
  • CyberForce：模拟重量/惯性，腕部最大 8.8N

各章节知识点
01

VR 系统架构与图形管线

  • 四大任务：读输入 → 更新状态 → 渲染 → 输出
  • 管线三阶段：应用（CPU）→ 几何（GPU）→ 光栅化（GPU）
  • Phong 光照模型：环境光 + 漫反射 + 镜面反射
  • 着色模型：Flat < Gouraud < Phong（质量递增）
02

光栅化阶段 + 优化

  • 反锯齿（Antialiasing）：像素细分子像素，取平均
  • 双缓冲（Double Buffering）：前/后缓冲区交换，消除闪烁
  • 瓶颈：CPU-limited / Transform-limited / Fill-limited
  • 应用阶段优化：更快 CPU/RAM/编译器
  • 几何阶段优化：减光源/减多边形
  • 光栅化优化：降分辨率/减多边形
03

触觉渲染管线

  • 碰撞检测阶段：加载物理属性，判断碰撞
  • 力计算阶段：Compute + Smooth + Map
  • 触觉计算阶段：叠加振动/温度等效果
  • 图形 vs 触觉频率：30–60fps vs 1KHz → 必须解耦
04

GPU 架构

  • GPU 优势：大规模数据并行，高带宽
  • CPU 优势：低延迟，控制密集型任务
  • GPU 增长率：~2.0×/年（CPU 为 ~1.5×/年）
  • Vertex Processor → Rasterizer → Fragment Processor
  • 内存：不支持直接寻址，仅向量寻址（1D/2D/3D）
05

分布式 VR 与多管线同步

  • 单用户：多并排显示；多 PC 局域网
  • 多用户：Client-Server / Peer-to-Peer / Hybrid
  • 三种同步方法：软件 + 帧缓冲 + 视频（最佳为三者结合）
06

PC 集群

  • 需求：大型拼接显示需独立管线，单 PC 不够
  • 优势：低成本；软件拼接
  • 限制：LAN 吞吐量是主要限制因素
  • WireGL：拦截 OpenGL 命令，按屏幕空间分发
  • 案例：Princeton（24台，6000×3000px）
